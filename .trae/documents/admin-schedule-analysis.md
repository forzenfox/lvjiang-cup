# 管理员赛程管理界面分析与优化建议

在您上次提出优化请求后，我已经对 `AdminSchedule` 及其子组件 (`MatchRow`, `ScheduleFilterBar`) 进行了深入重构。当前的实现已经相当成熟，具备了列表视图、高级筛选、行内编辑和淘汰赛自动晋级功能。

然而，为了进一步提升“直观性”和“管理效率”，我们可以在现有基础上进行更精细的优化。

## 1. 现状分析

### 优点
- **列表化管理**：相比卡片流，信息密度高，一屏能看更多。
- **筛选功能**：支持按赛段、状态、搜索队伍筛选，查找方便。
- **行内编辑**：无需跳转页面即可修改比分和状态。
- **自动逻辑**：已实现比分自动判定胜者、淘汰赛自动晋级。

### 潜在痛点 (Pain Points)
1.  **瑞士轮信息缺失**：在编辑瑞士轮比赛时，管理员只能看到 `round` 字段（如 "Day 1"），但无法直观设置或查看 `swissRecord` (0-0, 1-0 等) 和 `swissDay`。这对于我们刚刚优化的瑞士轮前台展示至关重要。
2.  **淘汰赛视觉断层**：管理员只能看到线性的比赛列表，无法直观感受到淘汰赛的树状结构关系（谁赢了去哪）。虽然有文字提示（`nextMatchId`），但不够直观。
3.  **缺乏批量操作**：如果有一天比赛全部延期，或者需要批量确认，目前只能逐个点击。
4.  **状态反馈不明显**：虽然有文字提示自动晋级，但没有明显的 UI 动画或高亮告知管理员“刚刚发生了自动晋级”。

## 2. 优化方案 (Optimization Plan)

### 2.1 增强 MatchRow 信息展示 (针对瑞士轮)
- **瑞士轮专用字段编辑**：
  - 在编辑模式下，如果赛段是 `swiss`，增加 `Swiss Round` (下拉: 0-0, 1-0...) 和 `Day` 的选择器。
  - 在查看模式下，显示瑞士轮的具体轮次标签（如 "1-0 组"）。

### 2.2 视觉优化与状态反馈
- **胜者高亮增强**：在查看模式下，获胜队伍的名字加粗并使用更醒目的颜色（如金色），甚至加上皇冠图标（目前已有 Trophy，可优化样式）。
- **晋级路径可视化提示**：对于淘汰赛，鼠标悬停在 `nextMatchId` 上时，高亮显示该比赛的去向（如果能做到列表内高亮最好，做不到则优化文字描述，如 "胜者 -> G5 (胜者组决赛)"）。

### 2.3 操作体验优化
- **智能默认值**：
  - 创建/修改比赛时，如果选择了 `elimination` 赛段，自动填充一些默认的 `round` 格式。
- **保存反馈**：保存成功后显示更明确的 Toast 提示（目前仅 `console.log` 或简单的 UI 更新）。

## 3. 实施步骤

1.  **修改 `MatchRow.tsx`**：
    - 增加对 `swissRecord` 和 `swissDay` 的编辑支持。
    - 优化查看模式下的元数据展示（区分显示瑞士轮轮次和淘汰赛轮次）。
2.  **优化 `AdminSchedule.tsx`**：
    - 引入 `sonner` 或类似的 Toast 组件，在自动晋级触发时给予管理员明确反馈（例如："已自动将 [Team Name] 晋级到 [Next Match]"）。
