# 瑞士轮晋级/淘汰名单管理方案对比

## 方案对比

| 维度 | 方案A：自动计算算法 | 方案B：管理员手动界面 |
|------|---------------------|----------------------|
| **实现复杂度** | 中高（需要开发算法+测试） | 中等（主要是UI开发） |
| **灵活性** | 低（规则固定） | 高（完全可控） |
| **错误风险** | 中（算法可能有bug） | 低（人工确认） |
| **使用便捷性** | 高（自动更新） | 中（需要手动操作） |
| **适用场景** | 规则明确的固定赛制 | 需要灵活调整的比赛 |
| **维护成本** | 高（算法需要维护） | 低（简单直观） |

---

## 推荐方案：混合方案（自动计算 + 管理员确认）

结合两种方案的优点，提供：
1. **自动计算** - 系统根据比赛结果自动计算推荐名单
2. **管理员确认** - 提供管理界面让管理员查看、调整、确认最终名单
3. **手动覆盖** - 特殊情况下管理员可以完全手动设置

---

## 混合方案详细计划

### 1. 项目概述

#### 目标
开发一个瑞士轮晋级/淘汰名单管理系统，包含自动计算和管理员手动调整功能。

#### 核心功能
- 根据比赛结果自动计算推荐晋级名单
- 提供管理界面让管理员查看和调整名单
- 支持完全手动模式（特殊情况）
- 管理员确认后名单生效

### 2. 数据流设计

```
比赛结果更新
    ↓
自动计算算法 → 推荐晋级名单
    ↓
管理员界面（显示推荐结果）
    ↓
管理员确认/调整
    ↓
生效的晋级名单
    ↓
前端展示
```

### 6. 核心算法逻辑

#### 自动计算算法

```typescript
function calculateSwissAdvancement(matches: Match[]): SwissAdvancementResult {
  // 1. 筛选已完成的瑞士轮比赛
  const finishedMatches = matches.filter(
    m => m.stage === 'swiss' && m.status === 'finished' && m.winnerId
  );

  // 2. 统计每支队伍战绩
  const records = calculateTeamRecords(finishedMatches);

  // 3. 分类队伍
  const winners2_0: string[] = [];
  const winners2_1: string[] = [];
  const oneTwoTeams: string[] = [];
  const eliminated0_3: string[] = [];

  for (const [teamId, record] of records) {
    if (record.wins === 2 && record.losses === 0) {
      winners2_0.push(teamId);
    } else if (record.wins === 2 && record.losses === 1) {
      winners2_1.push(teamId);
    } else if (record.wins === 1 && record.losses === 2) {
      oneTwoTeams.push(teamId);
    } else if (record.wins === 0 && record.losses === 3) {
      eliminated0_3.push(teamId);
    }
  }

  // 4. 处理1-2战绩组积分
  const {losersBracket, eliminated3rd} = processOneTwoTeams(
    finishedMatches, 
    oneTwoTeams
  );

  return {
    winners2_0,
    winners2_1,
    losersBracket,
    eliminated3rd,
    eliminated0_3
  };
}
```

### 7. 使用流程

#### 正常流程
1. 比赛结果更新
2. 系统自动计算推荐名单
3. 管理员查看推荐结果
4. 确认无误后点击"保存确认"
5. 名单生效并在前端展示

#### 调整流程
1. 管理员查看推荐结果
2. 发现需要调整（如特殊规则）
3. 拖拽调整队伍分类
4. 预览变更效果
5. 确认后保存

#### 手动流程
1. 管理员切换到"完全手动"模式
2. 手动设置所有分类
3. 保存生效

---

## 方案选择建议

**推荐采用混合方案**，原因：
1. 既保留了自动计算的便捷性
2. 又给予管理员完全的控制权
3. 适应各种特殊情况
4. 用户体验更好（有推荐但可调整）

如果资源有限，可以**优先实现方案B（管理员手动界面）**，因为：
- 实现更简单直接
- 完全满足当前需求
- 后期可以很容易地叠加自动计算功能
