# 删除自动晋级逻辑计划

## 背景
小型娱乐比赛不需要复杂的自动晋级功能，管理员可以手动配置淘汰赛对阵。需要删除相关代码以简化系统。

## 需要修改的文件清单

### 1. 类型定义文件
**文件**: `src/types/index.ts`
**操作**: 删除 Match 接口中的晋级关系字段
```typescript
// 删除以下字段（第40-43行）
nextMatchId?: string;           // 晋级后的下一场比赛ID
nextMatchSlot?: 'teamA' | 'teamB'; // 晋级后进入哪个位置
previousMatchIds?: string[];    // 来源比赛ID（用于败者组）
```

### 2. 后端逻辑文件
**文件**: `src/pages/admin/Schedule.tsx`
**操作**: 删除自动晋级检查逻辑（第43-72行）
- 保留比赛更新逻辑
- 删除 `Check for Auto-Advance` 代码块
- 删除相关的 toast 提示

### 3. UI 组件文件
**文件**: `src/pages/admin/components/MatchRow.tsx`
**操作**: 删除 nextMatchId 的显示逻辑（第277-284行）
- 删除查看模式下显示晋级关系的代码

### 4. Mock 数据文件
**文件**: `src/mock/data.ts`
**操作**: 删除所有比赛数据中的晋级关系字段
- 涉及比赛: elim-g1 到 elim-g8
- 删除字段: `nextMatchId`, `nextMatchSlot`, `previousMatchIds`

### 5. 文档文件
**文件**: `README.md`
**操作**: 
- 删除第18行中的"（支持自动晋级逻辑）"
- 删除第164-165行的字段说明
- 删除第187行的功能说明

**文件**: `docs/ADMIN_GUIDE.md`
**操作**:
- 删除第131-158行的"自动晋级机制"和"设置晋级关系"章节
- 删除第228-234行的"自动晋级没有生效怎么办"FAQ

## 实施步骤

1. **修改类型定义** (`src/types/index.ts`)
   - 删除 Match 接口中的三个字段

2. **删除后端逻辑** (`src/pages/admin/Schedule.tsx`)
   - 删除 handleMatchUpdate 中的自动晋级检查代码块

3. **简化 UI** (`src/pages/admin/components/MatchRow.tsx`)
   - 删除查看模式下显示 nextMatchId 的代码

4. **清理 Mock 数据** (`src/mock/data.ts`)
   - 删除所有淘汰赛比赛数据中的晋级关系字段

5. **更新文档** (`README.md`, `docs/ADMIN_GUIDE.md`)
   - 删除所有与自动晋级相关的描述

## 预期结果

- 淘汰赛管理界面更加简洁
- 管理员手动配置对阵，无需复杂的晋级关系设置
- 代码量减少，维护成本降低
- 文档描述与实际功能一致

## 注意事项

- 删除后，管理员需要手动为每轮比赛配置参赛队伍
- 不会影响现有的比赛数据展示功能
- 不会影响比赛状态管理和比分记录功能
