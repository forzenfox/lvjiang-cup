# 主页面赛事视频模块 PRD

## 1. 产品概览

### 1.1 项目背景
随着赛事内容的不断丰富，用户对于赛事相关视频的需求日益增长。为了提升用户体验，增强网站的内容吸引力，需要在主页面新增一个视频模块，展示赛事相关的视频内容。

### 1.2 目标用户
- **普通观众**：对赛事感兴趣，希望快速了解赛事相关视频内容
- **赛事粉丝**：关注特定赛事，希望方便地观看赛事回放、精彩集锦等内容

### 1.3 产品价值
- 提升主页面的内容丰富度和视觉吸引力
- 为用户提供便捷的赛事视频观看入口
- 增加网站的用户停留时间和页面浏览量
- 满足运营团队对赛事内容推广的需求

## 2. 核心功能

### 2.1 功能概览
| 功能模块 | 功能描述 |
|---------|--------|
| 视频轮播展示 | 左右滚动切换视频，最多展示5个视频 |
| B站视频嵌入 | 通过B站嵌入式播放器播放视频内容 |
| 后台管理配置 | 管理员通过后台页面配置视频链接 |
| 响应式设计 | 支持PC端和移动端适配 |

### 2.2 详细功能描述

#### 2.2.1 视频轮播展示
- **左右滚动**：用户可通过点击左右箭头或滑动屏幕切换视频
- **自动播放**：默认情况下，当前视频自动播放
- **视频预览**：每个视频卡片显示视频缩略图和标题
- **数量限制**：最多展示5个视频，超过5个时通过滚动查看

#### 2.2.2 B站视频嵌入
- **嵌入式播放器**：使用B站提供的嵌入式播放器代码
- **播放控制**：支持播放/暂停、音量调节、全屏等基本控制
- **清晰度选择**：根据B站播放器支持的清晰度自动适配
- **弹幕功能**：默认开启弹幕功能，与B站保持一致

#### 2.2.3 后台管理配置
- **视频链接管理**：管理员可添加、编辑、删除视频链接
- **排序功能**：可调整视频的展示顺序
- **状态管理**：可启用/禁用视频展示
- **链接验证**：验证B站链接的有效性

#### 2.2.4 响应式设计
- **PC端**：横向排列，展示多个视频卡片
- **移动端**：垂直排列，每次展示一个视频卡片，支持滑动切换
- **适配断点**：针对不同屏幕尺寸进行适配

## 3. 页面设计

### 3.1 设计风格
- **整体风格**：参考LPL官方页面设计，保持与现有网站风格一致
- **色彩方案**：使用网站主色调，确保视觉一致性
- **字体**：使用网站统一字体，保证品牌识别度

### 3.2 页面布局
- **位置**：主页面核心位置，用户进入网站后一眼可见
- **尺寸**：宽度与主页面一致，高度适中，确保不影响其他内容展示
- **布局**：
  - 顶部：模块标题（如"赛事精彩视频"）
  - 中部：视频轮播区域，支持左右滚动
  - 底部：查看更多链接（可选）

### 3.3 交互设计
- **滚动交互**：
  - PC端：点击左右箭头按钮切换视频
  - 移动端：左右滑动屏幕切换视频
- **播放交互**：点击视频卡片自动播放对应视频
- **悬停效果**：PC端鼠标悬停时视频卡片有轻微放大效果

### 3.4 视觉元素
- **视频卡片**：包含视频缩略图、标题、时长等信息
- **滚动指示器**：显示当前播放视频位置和总视频数量
- **控制按钮**：左右滚动箭头，样式与整体设计风格一致

## 4. 技术实现

### 4.1 前端实现
- **技术栈**：使用现有网站的前端技术栈
- **轮播组件**：实现左右滚动的轮播组件，支持触摸滑动
- **B站播放器集成**：使用B站提供的嵌入式播放器代码
  - 嵌入方式：使用iframe标签嵌入B站播放器
  - 参数配置：根据需要配置播放器参数，如自动播放、弹幕等

### 4.2 后端实现
- **视频管理API**：提供视频链接的增删改查接口
- **数据存储**：存储视频链接、标题、排序等信息
- **链接验证**：验证B站链接的有效性，确保播放器能正常加载

### 4.3 后台管理
- **管理界面**：在现有后台系统中添加视频管理模块
- **操作权限**：只有管理员角色可操作视频管理功能
- **操作流程**：
  1. 管理员登录后台
  2. 进入视频管理模块
  3. 添加/编辑/删除视频链接
  4. 调整视频排序
  5. 保存配置

## 5. 移动端适配

### 5.1 适配策略
- **响应式布局**：使用CSS媒体查询实现不同屏幕尺寸的适配
- **触摸优化**：针对移动设备的触摸操作进行优化
- **性能优化**：确保在移动设备上加载流畅

### 5.2 适配断点
- **PC端**：屏幕宽度 ≥ 1024px
- **平板端**：768px ≤ 屏幕宽度 < 1024px
- **移动端**：屏幕宽度 < 768px

### 5.3 移动端特殊处理
- **视频卡片**：移动端每次只展示一个视频卡片
- **滚动方式**：支持左右滑动切换视频
- **播放器尺寸**：根据屏幕宽度自动调整播放器大小
- **加载优化**：优先加载当前视频，其他视频延迟加载

## 7. 风险与应对策略

| 风险 | 应对策略 |
|------|--------|
| B站播放器加载失败 | 增加错误处理，显示备用内容 |
| 视频加载缓慢 | 实现懒加载，优化加载顺序 |
| 移动端性能问题 | 优化代码，减少不必要的资源加载 |
| 后台管理权限问题 | 严格控制操作权限，添加操作日志 |

## 8. 验收标准

1. **功能完整性**：所有核心功能正常实现
2. **视觉效果**：与设计稿一致，响应式布局适配不同设备
3. **性能表现**：页面加载速度快，视频播放流畅
4. **兼容性**：支持主流浏览器和移动设备
5. **管理功能**：后台管理功能正常，操作便捷
6. **用户体验**：交互流畅，操作简单直观
