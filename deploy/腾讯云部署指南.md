# 腾讯云服务器快速部署指南

> 专为腾讯元等国内服务器优化的部署流程

## 问题说明

在腾讯云服务器上部署时，从 GitHub Container Registry 拉取镜像速度很慢（通常只有 0.1-1 MB/s），原因是：
- 默认镜像源服务器位于海外
- 国际网络带宽受限

## 解决方案（3 步完成）

### 第 1 步：配置 Docker 镜像加速（5 分钟）

**使用一键脚本（推荐）：**

```bash
curl -fsSL https://raw.githubusercontent.com/forzenfox/lvjiang-cup/main/deploy/setup-docker-mirror.sh | sudo bash
```

**或手动配置：**

```bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com",
    "https://docker.m.daocloud.io"
  ]
}
EOF

sudo systemctl daemon-reload
sudo systemctl restart docker
```

**验证加速效果：**

```bash
docker info | grep -A 5 "Registry Mirrors"
# 应该显示配置的镜像地址
```

### 第 2 步：部署应用（10 分钟）

```bash
# 创建部署目录
mkdir -p /opt/lvjiang-cup/deploy
cd /opt/lvjiang-cup/deploy

# 下载统一部署脚本
curl -fsSL https://raw.githubusercontent.com/forzenfox/lvjiang-cup/main/deploy/setup.sh -o setup.sh
chmod +x setup.sh

# 运行部署脚本
./setup.sh
```

### 第 3 步：配置 Nginx Proxy Manager（5 分钟）

1. 访问管理界面：`http://服务器 IP:8181`
2. 登录（默认账号）：
   - 邮箱：`admin@example.com`
   - 密码：`changeme`
3. 首次登录会要求修改密码
4. 添加代理主机（参考下文配置）

## Nginx Proxy Manager 配置

### 前端代理配置

| 字段 | 值 |
|------|-----|
| Domain Names | `your-domain.com` |
| Scheme | `http` |
| Forward Hostname/IP | `服务器内网 IP` |
| Forward Port | `3001` |
| Cache Assets | ✅ 勾选 |
| Websockets Support | ✅ 勾选 |

### 后端代理配置

| 字段 | 值 |
|------|-----|
| Domain Names | `api.your-domain.com` |
| Scheme | `http` |
| Forward Hostname/IP | `服务器内网 IP` |
| Forward Port | `3000` |
| Websockets Support | ✅ 勾选 |

### SSL 证书配置

1. 在代理主机编辑页面，点击 **SSL** 标签
2. 选择 **Request a new SSL Certificate**
3. 输入邮箱
4. 同意 Let's Encrypt Terms
5. 保存

## 常见问题

### Q1: 镜像拉取仍然很慢？

**检查点：**
1. Docker 配置是否正确
2. 镜像源地址是否可访问

**解决方法：**

```bash
# 1. 检查配置文件
cat /etc/docker/daemon.json

# 2. 重启 Docker
sudo systemctl restart docker

# 3. 测试其他镜像源
# 编辑 /etc/docker/daemon.json 更换镜像源地址
```

### Q2: 无法访问 GitHub Raw 内容？

**解决方法：**

使用镜像地址下载文件：

```bash
# 使用 FastGit
curl -fsSL https://raw.fastgit.org/forzenfox/lvjiang-cup/main/deploy/setup.sh -o setup.sh

# 或使用其他 CDN 镜像
curl -fsSL https://cdn.jsdelivr.net/gh/forzenfox/lvjiang-cup@main/deploy/setup.sh -o setup.sh
```

### Q3: Docker 网络创建失败？

**解决方法：**

```bash
# 1. 检查现有网络
docker network ls

# 2. 删除冲突网络（如果有）
docker network rm npm-network

# 3. 重新创建
docker network create npm-network
```

### Q4: 容器启动失败？

**排查步骤：**

```bash
# 1. 查看容器状态
docker compose ps

# 2. 查看容器日志
docker compose logs backend
docker compose logs frontend

# 3. 检查端口占用
netstat -tulpn | grep -E '3000|3001'

# 4. 检查配置文件
cat .env
cat config.js
```

## 性能优化建议

### 1. 系统层面

```bash
# 增加文件描述符限制
echo "fs.file-max = 1000000" | sudo tee -a /etc/sysctl.conf
echo "fs.inotify.max_user_instances = 8192" | sudo tee -a /etc/sysctl.conf
sudo sysctl -p
```

### 2. Docker 层面

```bash
# 优化 Docker 配置（已包含在加速脚本中）
{
  "max-concurrent-downloads": 10,
  "storage-driver": "overlay2",
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m",
    "max-file": "3"
  }
}
```

### 3. 网络层面

```bash
# 修改 DNS 为腾讯云 DNS
sudo tee /etc/resolv.conf <<-'EOF'
nameserver 183.60.83.19
nameserver 183.60.82.98
EOF
```

## 验证部署

```bash
# 1. 检查容器状态
docker compose ps

# 应该看到：
# NAME                STATUS         PORTS
# lvjiang-backend     Up (healthy)   3000/tcp
# lvjiang-frontend    Up (healthy)   3001/tcp

# 2. 检查服务健康状态
docker exec lvjiang-backend wget -q -O /dev/null http://127.0.0.1:3000/api/teams && echo "Backend OK"
docker exec lvjiang-frontend wget -q -O /dev/null http://127.0.0.1:3001/ && echo "Frontend OK"

# 3. 测试 API
curl http://localhost:3000/api/teams

# 4. 测试前端
curl http://localhost:3001/
```
