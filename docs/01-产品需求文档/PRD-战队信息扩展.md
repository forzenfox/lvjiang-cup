# 战队信息扩展功能 PRD

## 1. 文档概述

### 1.1 产品背景
当前战队管理系统中，战队信息和队员信息展示较为基础，无法满足赛事管理的专业化需求。需要扩展战队信息展示维度，提升参赛队伍的个性化配置能力，增强赛事观赏性和专业性。

### 1.2 产品目标
- 提升战队信息展示的完整性和专业性
- 支持战队个性化配置（图标、宣言）
- 完善队员信息展示，支持常用英雄等游戏相关信息
- 提供便捷的管理后台配置能力

### 1.3 目标用户
- **参赛战队**：需要完善展示战队和队员信息
- **赛事管理员**：需要配置和管理战队信息
- **赛事观众**：查看战队和队员详细信息

### 1.4 适用范围
- 前端：战队信息展示页面、管理后台配置页面
- 后端：战队信息 API、队员信息 API、文件上传 API

## 2. 功能需求详情

### 2.1 战队图标上传功能

#### 2.1.1 功能描述
支持战队上传自定义图标，用于战队信息卡片展示。

#### 2.1.2 功能要点
- **上传方式**：支持点击图标区域选择本地图片上传
- **格式限制**：仅支持 JPG、PNG 格式
- **大小限制**：单张图片不超过 2MB
- **分辨率要求**：
  - 建议分辨率：512x512 px
  - 最小分辨率：256x256 px
  - 最大分辨率：1024x1024 px
- **自动处理**：
  - 前端上传前进行格式和大小校验
  - 后端接收后进行分辨率压缩处理（如超出范围）
  - 生成缩略图（128x128 px）用于列表展示

#### 2.1.3 交互说明
**管理后台：**
1. 点击战队图标区域，弹出文件选择器
2. 选择图片后，前端预览展示
3. 确认上传后，显示上传进度
4. 上传成功后，展示新图标
5. 提供"恢复默认"按钮，可重置为系统默认图标

**前端展示：**
- 战队信息卡片左上角展示战队图标
- 图标为方形完整展示，不进行裁剪
- 未上传时显示默认图标（战队首字母或占位图）

#### 2.1.4 边界条件
- 上传失败时显示错误提示（"图片上传失败，请重试"）
- 格式不支持时提示（"仅支持 JPG/PNG 格式图片"）
- 图片过大时提示（"图片大小不能超过 2MB"）
- 分辨率不满足时自动压缩并提示用户

### 2.2 参赛宣言功能

#### 2.2.1 功能描述
将原有"战队简介"字段修改为"参赛宣言"，突出赛事竞技属性。

#### 2.2.2 字段规格
- **字段名称**：battle_cry（参赛宣言）
- **数据类型**：字符串
- **长度限制**：最多 100 个字符（含标点符号）
- **必填性**：非必填
- **默认值**：空

#### 2.2.3 交互说明
**管理后台：**
1. 文本输入框，支持多行输入
2. 实时显示字符计数（格式：当前字符数/100）
3. 超出限制时禁止继续输入并提示
4. 支持清空操作

**前端展示：**
- 展示位置：战队信息卡片中部，战队名称下方
- 展示样式：斜体展示，增加引号装饰
- 未填写时不显示该字段（不显示空行）

#### 2.2.4 示例
```
示例 1："剑锋所指，所向披靡！"
示例 2："为了荣耀，为了梦想！"
示例 3："峡谷争锋，谁与争锋！"
```

### 2.3 战队成员信息扩展

#### 2.3.1 功能描述
扩展战队成员信息展示，点击队员栏弹出详情弹框，展示完整的队员信息。

#### 2.3.2 触发方式
- 用户点击战队信息卡片中的"队员"区域
- 弹出模态框展示所有队员详情

#### 2.3.3 交互说明
**触发入口：**
- 战队信息卡片中显示队员数量（如："队员 5 人"）
- 该区域支持点击，鼠标悬停时显示手型指针
- 点击后弹出"战队成员详情"弹框

**弹框设计：**
- 弹框标题：战队名称 + "成员详情"（如："EDG 战队成员详情"）
- 弹框宽度：800px（桌面端）、90% 宽度（移动端）
- 最大高度：600px，内容超出可滚动
- 关闭方式：点击右上角 X 按钮或弹框外区域

- **英雄数据来源**：英雄联盟官方英雄数据
- **展示形式**：英雄头像小图标（40x40 px）
- **交互**：鼠标悬停显示英雄名称

#### 2.4.4 队员卡片布局

**弹框内每个队员的展示布局：**


**样式要求：**
- 头像：方形完整展示，64x64 px，使用斗鱼用户头像外链接
- 昵称：加粗展示，16px 字体
- 队长标识：金色徽章图标，仅在 is_captain=true 时显示
- 游戏 ID：普通字体，14px，灰色
- 位置：图标 + 文字，14px
- 评分：星级展示 + 数字，14px，金色
- 直播间：显示直播间号，可点击跳转斗鱼直播页面，14px，蓝色链接色
- 简介：斜体，12px，最多显示 2 行，超出省略
- 常用英雄：横向排列，显示中文名称，12px，间距 8px

**常用英雄头像展示（悬停时）：**
- 当用户鼠标悬停在英雄名称上时，显示英雄头像小浮层
- 头像来源：使用腾讯官方 CDN，参考 `英雄头像展示技术方案文档.md`
- CDN 地址规则：`https://game.gtimg.cn/images/lol/act/img/champion/{英文名}.png`
- 实现方式：通过英雄中文名称查找对应英文名，拼接 CDN URL
- 浮层尺寸：80x80 px，圆角边框，阴影效果

### 2.5 管理后台配置功能

#### 2.5.1 功能描述
管理员可通过管理后台对战队信息和队员信息进行完整配置。

#### 2.5.2 配置页面结构

**页面布局：**
```
┌─────────────────────────────────────────┐
│ 战队信息管理                            │
├─────────────────────────────────────────┤
│ 基本信息区                              │
│ ┌─────────────────────────────────┐    │
│ │ [图标] 战队名称：EDG            │    │
│ │        参赛宣言：xxxx           │    │
│ └─────────────────────────────────┘    │
│                                         │
│ 队员管理区                              │
│ ┌─────────────────────────────────┐    │
│ │ 队员列表（支持增删改查）        │    │
│ │ + 添加队员                      │    │
│ └─────────────────────────────────┘    │
│                                         │
│ [保存] [取消]                           │
└─────────────────────────────────────────┘
```

#### 2.5.3 战队信息配置

**可配置项：**
1. **战队图标**
   - 上传组件，支持拖拽上传
   - 显示当前图标预览
   - 支持删除/更换

2. **战队名称**
   - 文本输入框
   - 必填项
   - 最多 20 个字符

3. **参赛宣言**
   - 多行文本输入框
   - 非必填
   - 实时字符计数

#### 2.5.4 队员管理功能

**队员列表展示：**
- 表格形式展示所有队员
- 列信息：头像、昵称、游戏 ID、位置、评分、队长标识、直播间、操作
- 支持按位置筛选
- 支持按评分排序
- 支持按是否队长筛选
- **默认排序**：按照位置固定顺序排序（上单→打野→中单→射手→辅助）
- **不支持自定义排序**

**添加队员：**
1. 点击"添加队员"按钮
2. 弹出队员编辑表单
3. 填写完整信息（所有字段）
4. 保存后添加到列表

**编辑队员：**
1. 点击操作列"编辑"按钮
2. 弹出队员编辑表单（回显当前数据）
3. 修改后保存

**删除队员：**
1. 点击操作列"删除"按钮
2. 二次确认（"确定要删除该队员吗？"）
3. 确认后从列表移除

**队员排序：**
- **不支持拖拽排序**
- **固定按照位置自动排序**（上单→打野→中单→射手→辅助）
- 同一位置内按添加时间排序（先添加的在前）

#### 2.5.5 队员编辑表单

**表单字段：**

| 字段 | 组件类型 | 必填 | 说明 |
|------|----------|------|------|
| 头像 | 文本输入 | 否 | 斗鱼用户头像外链接 URL |
| 昵称 | 文本输入 | 是 | 最多 20 字符 |
| 游戏 ID | 文本输入 | 是 | 最多 30 字符 |
| 位置 | 下拉选择 | 是 | 5 个位置选项（带图标） |
| 个人简介 | 多行文本 | 否 | 最多 200 字符 |
| 常用英雄 | 英雄选择器 | 否 | 多选，最多 5 个，仅显示中文名称 |
| 评分 | 数字输入/滑块 | 否 | 0-100 整数，默认 60 |
| 是否队长 | 单选框 | 否 | true/false，同一战队只能有一个队长 |
| 直播间链接 | 文本输入 | 否 | 斗鱼直播间完整 URL |

**英雄选择器设计：**
- 弹出面板展示所有英雄中文名称列表（带称号，如"疾风剑豪·亚索"）
- 支持搜索（按英雄中文名称，支持模糊匹配，如输入"亚索"可匹配）
- 不需要位置筛选功能
- 已选英雄高亮显示（蓝色边框 + 对勾标记）
- 显示已选数量（当前数/5）
- 英雄数据来源：参考 `英雄头像展示技术方案文档.md` 中的英雄映射表
- 点击英雄名称可选中/取消选中
- 支持批量选择（点击选择器内的"确认"按钮关闭弹框）

**英雄数据映射表：**
```json
{
  "Yasuo": "疾风剑豪·亚索",
  "Zed": "影流之主·劫",
  "Ryze": "流浪法师·瑞兹",
  // ... 完整列表参考 `英雄头像展示技术方案文档.md`
}
```

#### 2.5.6 数据校验

**前端校验：**
- 必填字段校验
- 格式校验（图片格式、大小）
- 长度校验（字符数限制）
- 实时校验，错误时显示错误信息

**后端校验：**
- 数据完整性校验
- 业务规则校验（如战队最多 10 人）
- 返回明确错误信息

#### 2.5.7 保存逻辑

**保存按钮：**
- 保存前进行完整数据校验
- 校验通过后才发起保存请求
- 保存时按钮显示 loading 状态
- 保存成功后提示"保存成功"并返回列表
- 保存失败时提示错误原因，保留当前编辑状态

**取消按钮：**
- 点击后提示"确定要放弃修改吗？"
- 确认后返回列表页，不保存任何修改

## 3. 数据结构设计

### 3.1 战队信息表（teams）

```json
{
  "id": "integer, 主键，自增",
  "name": "string, 战队名称，最多 20 字符，必填",
  "logo_url": "string, 战队图标 URL，可空",
  "logo_thumbnail_url": "string, 缩略图 URL，可空",
  "battle_cry": "string, 参赛宣言，最多 100 字符，可空",
  "created_at": "datetime, 创建时间",
  "updated_at": "datetime, 更新时间",
  "created_by": "integer, 创建人 ID",
  "status": "integer, 状态：0-禁用，1-启用"
}
```

### 3.2 队员信息表（team_members）

```json
{
  "id": "integer, 主键，自增",
  "team_id": "integer, 外键，关联 teams.id",
  "user_id": "integer, 关联用户表 ID",
  "avatar_url": "string, 头像 URL（斗鱼用户头像外链接），可空",
  "nickname": "string, 昵称，最多 20 字符，必填",
  "game_id": "string, 游戏 ID，最多 30 字符，必填",
  "position": "string, 位置枚举：TOP/JUNGLE/MID/ADC/SUPPORT，必填",
  "bio": "string, 个人简介，最多 200 字符，可空",
  "champion_pool": "json, 常用英雄中文名称列表，如 [\"亚索\", \"劫\", \"瑞兹\"]",
  "rating": "integer, 评分，0-100 整数，默认 60，可空",
  "is_captain": "boolean, 是否队长，默认 false，可空",
  "live_url": "string, 直播间链接（斗鱼 URL），可空",
  "sort_order": "integer, 排序号，数字越小越靠前",
  "created_at": "datetime, 创建时间",
  "updated_at": "datetime, 更新时间"
}
```

### 3.3 API 接口设计

```
GET /api/teams/{id}
Response:

  "code": 0,
  "data": {
    "id": 1,
    "name": "EDG",
    "logo_url": "/uploads/teams/1/logo.png",
```

```
PUT /api/teams/{id}
Request:
{
```

## 5. 边界条件与异常处理

### 5.1 图片上传

**上传失败：**
- 网络错误：提示"网络异常，请重试"
- 服务器错误：提示"上传失败，请稍后重试"
- 格式错误：提示"仅支持 JPG/PNG 格式"
- 大小超限：提示"图片大小不能超过 2MB"

**处理方案：**
- 提供重试按钮
- 保留用户已填写的其他信息
- 支持更换已上传的图片

### 5.2 数据加载

**加载失败：**
- 战队信息加载失败：显示"数据加载失败，请刷新重试"
- 队员信息加载失败：弹框内显示错误提示
- 英雄数据加载失败：常用英雄显示为文字列表

**加载中：**
- 显示 loading 动画
- 禁止重复点击
- 超时提示（超过 10 秒）

### 5.3 数据校验

**前端校验失败：**
- 实时校验，错误字段高亮
- 显示具体错误信息
- 禁止提交

**后端校验失败：**
- 返回明确错误码和错误信息
- 前端弹窗提示
- 保留用户输入

### 5.4 并发处理

**多人同时编辑：**
- 后编辑者覆盖先编辑者（乐观锁）
- 提示"数据已被其他管理员修改"
- 建议刷新后重新编辑

**队员数量限制：**
- 单战队最多 10 名队员
- 达到上限时禁止添加
- 提示"战队人数已达上限"
