## 2. 项目背景与目标

### 2.1 背景
当前战队模块的UI设计需要参考官方世界赛的视觉风格进行优化，提升用户体验和界面美观度。

### 2.2 目标
- 仿照官方世界赛战队模块的UI风格
- 优化信息展示层次，提升用户浏览效率
- 改善交互体验，使战队和队员信息的获取更加直观便捷
- 实现全设备响应式适配

## 3. 参考设计

### 3.1 官方世界赛战队模块
- **主页面**：采用深色背景，战队图标以网格形式展示，每个卡片仅包含战队Logo和战队名称
- **弹框设计**：点击战队后显示战队成员列表，顶部展示战队Logo和名称，下方展示选手信息表格

## 4. 功能需求详述

### 4.1 主页面战队列表展示

#### 4.1.1 页面布局
- 采用网格布局展示战队卡片
- 每个战队卡片仅包含：
  - 战队Logo（队标）
  - 战队名称（队名）
- 卡片尺寸：PC端 200x200px，平板端 180x180px，手机端 150x150px
- 卡片间距：PC端 24px，平板端 16px，手机端 12px
- 排列方式：PC端 5列，平板端 4列，手机端 2列

#### 4.1.2 交互设计
- 鼠标悬停效果：卡片背景色变化，添加轻微阴影
- 点击交互：点击战队卡片，弹出战队成员列表弹框
- 加载状态：添加loading动画效果

### 4.2 战队成员列表弹框

#### 4.2.1 弹框设计
- 弹框样式：居中显示，半透明遮罩层
- 弹框尺寸：PC端 800x600px，平板端 700x500px，手机端 90%屏幕宽度
- 弹框内容：
  - 顶部区域：战队Logo和战队名称
  - 成员列表：选手ID、选手位置
  - **移除官方设计中的选手照片**

#### 4.2.2 成员列表设计
- 列表项：每行显示一名选手信息
- 每行包含：
  - 左侧：选手ID
  - 右侧：选手位置图标
- 行高：PC端 50px，平板端 45px，手机端 40px
- 分隔线：浅灰色分隔线
- 交互：点击成员列表项，右侧展开队员详情抽屉

#### 4.2.3 交互设计
- 打开弹框：点击战队卡片，弹框从中间渐显
- 关闭弹框：点击遮罩层或右上角关闭按钮
- 加载状态：弹框内显示loading动画

### 4.3 队员详情抽屉

#### 4.3.1 抽屉设计
- 显示位置：弹框右侧（PC端/平板端），手机端从底部弹出
- 抽屉尺寸：PC端/平板端 宽度400px，手机端 100%屏幕宽度，高度70%
- 内容区域：与现有队员详情弹框内容保持一致
  - 选手基本信息（ID、位置、国籍等）
  - 选手数据统计
  - 选手历史战绩
  - 选手简介

#### 4.3.2 交互设计
- 打开抽屉：点击成员列表项，抽屉从右侧滑入
- 关闭抽屉：点击抽屉内关闭按钮或点击弹框其他区域
- 切换队员：点击其他队员，直接切换抽屉内容，无需关闭再打开

### 4.4 响应式适配方案

#### 4.4.1 PC端（≥1200px）
- 战队卡片：5列网格布局，卡片200x200px
- 战队弹框：800x600px，居中显示
- 队员抽屉：右侧滑出，宽度400px

#### 4.4.2 平板端（768px-1199px）
- 战队卡片：4列网格布局，卡片180x180px
- 战队弹框：700x500px，居中显示
- 队员抽屉：右侧滑出，宽度350px

#### 4.4.3 手机端（≤767px）
- 战队卡片：2列网格布局，卡片150x150px
- 战队弹框：90%屏幕宽度，高度自适应
- 队员抽屉：从底部滑出，宽度100%，高度70%

## 5. 技术实现方案

### 5.1 前端技术栈
- React + TypeScript
- 使用Ant Design或类似UI组件库
- CSS Module或Styled Components

### 5.2 核心组件
- `TeamGrid`：战队网格列表组件
- `TeamCard`：战队卡片组件
- `TeamModal`：战队成员列表弹框组件
- `PlayerDrawer`：队员详情抽屉组件

### 5.3 数据接口
- 战队列表接口：`GET /api/teams`
- 战队详情接口：`GET /api/teams/{teamId}`
- 队员详情接口：`GET /api/players/{playerId}`

## 6. 验收标准

### 6.1 功能验收
- [ ] 主页面正确展示战队图标和队名
- [ ] 点击战队卡片能正确弹出成员列表
- [ ] 成员列表不包含选手照片
- [ ] 点击队员能正确打开详情抽屉
- [ ] 抽屉内容与现有队员详情弹框一致

### 6.2 交互验收
- [ ] 所有交互反馈及时流畅
- [ ] 动画效果自然平滑
- [ ] 关闭操作符合用户预期

### 6.3 响应式验收
- [ ] PC端展示和交互正常
- [ ] 平板端展示和交互正常
- [ ] 手机端展示和交互正常
- [ ] 各断点过渡自然

### 6.4 性能验收
- [ ] 页面加载时间≤2秒
- [ ] 动画帧率≥60fps
- [ ] 内存占用合理

## 7. 风险与注意事项

### 7.1 技术风险
- 响应式适配可能影响布局稳定性
- 抽屉与弹框的层级关系需要妥善处理
- 移动端手势操作需要考虑

### 7.2 设计风险
- 与官方风格的相似度需要把握
- 移除选手照片后信息密度需要平衡

### 7.3 兼容性风险
- 需要测试主流浏览器兼容性
- 移动端不同设备的适配测试
