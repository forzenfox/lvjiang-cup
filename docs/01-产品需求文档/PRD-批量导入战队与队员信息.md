# 批量导入战队与队员信息 PRD

## 1. 文档概述

### 1.1 产品背景

当前驴酱杯赛事网站的管理后台中，战队和队员信息只能逐个手动创建和编辑。每支战队需要填写战队名称、队标、参赛宣言，以及5名队员（上单、打野、中单、ADC、辅助）各自的昵称、游戏ID、头像、评分、常用英雄等信息。当参赛战队数量较多（最多16支）时，管理员需要反复操作，流程繁琐、效率低下，且容易出错。

为简化管理员操作、提升效率，需要支持通过 Excel 文件批量导入战队信息和队员信息，使管理员能够一次性完成多支战队及其队员数据的录入。

### 1.2 产品目标

- 降低管理员录入战队和队员信息的操作成本，提升管理效率
- 支持 Excel 模板下载 → 填写 → 上传导入的标准化流程
- 提供完善的导入校验和错误反馈机制，确保数据质量
- 兼容现有手动管理模式，批量导入与手动编辑互不冲突

### 1.3 目标用户

- **赛事管理员**：需要快速录入大量战队和队员信息

### 1.4 适用范围

- 前端：管理后台战队管理页面（新增批量导入入口）
- 后端：新增批量导入 API 接口、Excel 解析与校验逻辑

***

## 2. 功能需求详情

### 2.1 Excel 模板下载

#### 2.1.1 功能描述

管理员可下载标准 Excel 导入模板，模板中包含所有可导入字段的列头和示例数据，以及填写说明。

#### 2.1.2 模板设计

**文件格式**：`.xlsx`

**Sheet 名称**：
- `战队与队员信息导入`：主数据填写Sheet
- `英雄列表（参考）`：英雄联盟英雄名称参考Sheet

**数据源**：英雄列表Sheet 数据来源于 `lol-champion-map.json` 文件，该文件可通过 `scripts/fetch-champions.mjs` 脚本从 Riot Data Dragon API 获取。生产部署时，前后端可挂载服务器磁盘上同一个英雄数据文件，确保数据一致性。

**设计原则**：

- 每支战队占 **5 行**数据（对应5个位置：上单、打野、中单、ADC、辅助）
- 战队信息（名称、队标URL、参赛宣言）**只在第1行填写**，后续4行队员行对应同一战队，该列留空
- 后端解析时，同一战队名称的连续5行数据视为一个战队的队员列表
- 若战队只有4名队员或更少，剩余位置行留空即可

**列定义**：

| 列序 | 列名（中文）  | 列名（英文标识）               | 字段类型       | 必填        | 说明                 | 示例                             |
| -- | ------- | ---------------------- | ---------- | --------- | ------------------ | ------------------------------ |
| A  | 战队名称    | team\_name             | 文本         | **是**（首行） | 最多20字符，不可重复；队员行可留空 | 龙之队                            |
| B  | 队标URL   | team\_logo\_url        | 文本         | 否（首行）     | 仅支持外链URL格式；队员行留空   | <https://example.com/logo.png> |
| C  | 参赛宣言    | team\_battle\_cry      | 文本         | 否（首行）     | 最多100字符；队员行留空      | 峡谷争锋，谁与争锋！                     |
| D  | 位置      | position               | **枚举（下拉）** | **是**     | 上单/打野/中单/ADC/辅助    | 上单                             |
| E  | 队员昵称    | member\_nickname       | 文本         | 否         | 最多20字符             | 小明                             |
| F  | 队员游戏ID  | member\_game\_id       | 文本         | 否         | 最多30字符             | Dragon Top                     |
| G  | 队员头像URL | member\_avatar\_url    | 文本         | 否         | 仅支持外链URL格式         | <https://example.com/top.png>  |
| H  | 评分      | member\_rating         | 整数         | 否         | 0-100，默认60         | 85                             |
| I  | 是否队长    | member\_is\_captain    | **枚举（下拉）** | 否         | 是/否，默认否            | 是                              |
| J  | 实力等级    | member\_level          | **枚举（下拉）** | 否         | S/A/B/C/D，不填则无等级   | A                              |
| K  | 常用英雄    | member\_champion\_pool | 文本         | 否         | 中文名称，逗号分隔，最多5个     | 疾风剑豪,离群之刺,影哨                   |

| M  | 个人简介    | member\_bio            | 文本         | 否         | 最多200字符            | 热爱电竞，追求梦想                      |

**模板列设置说明**：

- **位置列（D列）**：设置数据有效性为下拉列表，值：`上单,打野,中单,ADC,辅助`
- **是否队长列（I列）**：设置数据有效性为下拉列表，值：`是,否`
- **实力等级列（J列）**：设置数据有效性为下拉列表，值：`S,A,B,C,D`

**英雄列表 Sheet（第二个 Sheet）**：

Sheet 名称：`英雄列表（参考）`

| 列序 | 列名 | 说明 |
|------|------|------|
| A | 英文ID | 英雄英文标识，如 `Yasuo` |

- `id` → 英文ID

- `title` → 中文称号

- 管理员在填写"常用英雄"列时，可切换到此 Sheet 查看英雄的中文名称和称号

#### 2.1.3 交互说明

1. 在战队管理页面顶部操作栏增加"下载模板"按钮
2. 点击后直接下载 `.xlsx` 文件，文件名格式：`驴酱杯_战队导入模板_YYYYMMDD.xlsx`
3. 模板包含两个 Sheet：
   - `战队与队员信息导入`：主数据填写Sheet，包含示例数据
   - `英雄列表（参考）`：英雄联盟英雄信息参考，方便填写"常用英雄"列时查询
4. 管理员可删除示例数据后填写实际数据

### 2.3 导入模式

**统一采用「覆盖模式」**：

- **不存在的战队**：直接插入新数据
- **已存在的战队**（以战队名称为唯一标识）：使用导入数据整体覆盖原数据（全量替换战队信息和队员列表）
- 覆盖时**保留原战队ID**，不破坏关联的比赛数据
- 导入结果中明确标注哪些是新增、哪些是覆盖

- 文件名格式：`驴酱杯_导入错误报告_YYYYMMDD_HHmmss.xlsx`

### 2.5 导入后手动补充操作

#### 2.5.1 功能描述

批量导入支持大多数字段，但以下字段由于技术限制只能通过手动操作：

| 字段       | 原因              | 手动操作方式                    |
| -------- | --------------- | ------------------------- |
| 队标本地上传   | Excel 不支持嵌入图片文件 | 战队管理 → 编辑战队 → 上传队标        |
| 队员头像本地上传 | Excel 不支持嵌入图片文件 | 战队管理 → 展开战队 → 编辑队员 → 上传头像 |

#### 2.5.2 操作指引

导入结果反馈中，明确提示哪些战队/队员使用了外链URL，管理员可：

1. 在战队管理页面找到对应战队
2. 展开战队详情，进入编辑模式
3. 通过上传组件替换队标或队员头像为本地文件
4. 保存修改

## 3. 数据结构设计

### 3.1 Excel 导入数据映射

Excel 中的数据将映射到现有的数据库表结构，不新增数据表。

**解析规则**：

- Excel 数据按"战队名称"列分组
- 战队名称非空的行视为该战队的"首行"，包含战队基础信息
- 战队名称为空但位置非空的行，视为该战队的队员行
- 同一战队名称的连续多行（最多5行）组成该战队的队员列表
- 位置枚举映射：`上单`→`TOP`，`打野`→`JUNGLE`，`中单`→`MID`，`ADC`→`ADC`，`辅助`→`SUPPORT`

**战队表（teams）映射**：

| Excel 列               | 数据库字段       | 映射规则        |
| --------------------- | ----------- | ----------- |
| team\_name（首行）        | name        | 直接映射        |
| team\_logo\_url（首行）   | logo        | 直接映射（外链URL） |
| team\_battle\_cry（首行） | battle\_cry | 直接映射        |

**队员表（team\_members）映射**：

| Excel 列                | 数据库字段          | 映射规则                                             |
| ---------------------- | -------------- | ------------------------------------------------ |
| position               | position       | 枚举映射（上单→TOP等）                                    |
| member\_nickname       | nickname       | 直接映射                                             |
| member\_game\_id       | game\_id       | 直接映射                                             |
| member\_avatar\_url    | avatar\_url    | 直接映射（外链URL）                                      |
| member\_rating         | rating         | 直接映射，默认60                                        |
| member\_is\_captain    | is\_captain    | "是"→1，"否"→0，默认0                                  |
| member\_level          | level          | 直接映射枚举值 S/A/B/C/D                                |
| member\_champion\_pool | champion\_pool | 中文名称转英文名数组，如 `["Yasuo","Akali"]`                 |
| member\_live\_room     | live\_url      | 拼接斗鱼前缀，如 `123456`→`https://www.douyu.com/123456` |
| member\_bio            | bio            | 直接映射，最多200字符                                     |

### 3.2 仅支持本地文件上传的字段

以下字段**不支持通过 Excel 导入外链 URL**，仅支持在管理界面手动上传本地文件：

| 字段       | 原因              | 手动操作方式                    |
| -------- | --------------- | ------------------------- |
| 队标本地上传   | Excel 不支持嵌入图片文件 | 战队管理 → 编辑战队 → 上传队标        |
| 队员头像本地上传 | Excel 不支持嵌入图片文件 | 战队管理 → 展开战队 → 编辑队员 → 上传头像 |

**说明**：如果导入时填写了队标URL或头像URL（外链），系统会正常保存该外链地址。如需替换为本地文件，导入后在管理界面手动上传即可。

### 3.3 字段导入处理规则

#### 常用英雄（member\_champion\_pool）

- **填写格式**：中文英雄名称，逗号分隔，如 `疾风剑豪,离群之刺,影哨`
- **识别规则**：支持中文名称（name）和中文称号（title）两种写法
  - `疾风剑豪` → `Yasuo`
  - `亚托克斯` → `Aatrox`
  - `九尾妖狐,阿狸` → `Ahri`（title 或 name 均可匹配）
- **转换存储**：统一转换为英文名（id）存入数据库，如 `["Yasuo", "Akali", "Akshan"]`
- **校验规则**：最多5个英雄，名称必须能匹配到英雄联盟英雄库
- **错误示例**：`疾风剑豪,未知英雄` → 提示"未知英雄不在英雄库中"

#### 实力等级（member\_level）

- **填写格式**：枚举值 S/A/B/C/D 或留空
- **处理规则**：直接存储枚举值，与评分字段**无联动关系**
- **校验规则**：只能是 S/A/B/C/D 之一

#### 直播间号（member\_live\_room）

- **填写格式**：纯数字斗鱼房间号，如 `123456`
- **自动处理**：导入时自动拼接斗鱼链接前缀 `https://www.douyu.com/`
- **校验规则**：必须是纯数字，如填写 `123456` → 存储为 `https://www.douyu.com/123456`
- **错误示例**：`https://www.douyu.com/123456` → 提示"直播间号只需填写纯数字"

#### 个人简介（member\_bio）

- **填写格式**：纯文本，最多200字符
- **校验规则**：长度不超过200字符

## 4. API 接口设计

### 4.1 下载导入模板

```
GET /admin/teams/import/template
Authorization: Bearer <token>

Response:
  Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
  Content-Disposition: attachment; filename="驴酱杯_战队导入模板_20260415.xlsx"
  Body: Excel 文件二进制流（包含2个Sheet）
```

**模板生成逻辑**：
1. 检查本地是否存在已生成的模板文件（如 `/data/team-import-template.xlsx`）
2. 若模板文件存在且未过期（可设置版本号或过期时间），直接读取返回
3. 若模板文件不存在或已过期：
   - 读取 `lol-champion-map.json` 英雄数据文件
   - 生成 Sheet 1：`战队与队员信息导入`（包含列头和示例数据）
   - 生成 Sheet 2：`英雄列表（参考）`（从英雄数据转换而来，包含英文ID、中文名称、中文称号三列）
   - 保存生成的模板文件到本地（如 `/data/team-import-template.xlsx`）
4. 返回 Excel 文件流

**模板更新策略**：
- 模板内容相对稳定，英雄数据更新时需同步更新模板
- 可通过以下方式触发模板更新：
  - 手动调用更新接口 `POST /admin/teams/import/template/refresh`
  - 英雄数据文件发生变化时自动更新
  - 或每次下载时实时生成（数据量小，性能影响可忽略）

## 6. 边界条件与异常处理

### 6.1 数据冲突处理

| 场景               | 处理方式                 |
| ---------------- | -------------------- |
| 战队名称在文件内重复       | 校验阶段报错，不执行导入         |
| 战队名称与数据库重复（覆盖模式） | 使用导入数据整体覆盖原战队信息和队员数据 |
| 同一战队设置多名队长       | 校验阶段报错，不执行导入         |
| 导入后战队总数超过16      | 校验阶段报错，不执行导入         |

### 6.2 文件异常处理

| 场景           | 处理方式                        |
| ------------ | --------------------------- |
| 文件格式不是 .xlsx | 前端校验拦截，提示"仅支持 .xlsx 格式"     |
| 文件损坏无法解析     | 后端返回错误，提示"文件解析失败，请检查文件是否损坏" |
| 文件内容为空（只有列头） | 后端返回错误，提示"文件中没有有效数据"        |
| 文件大小超过5MB    | 前端校验拦截，提示"文件大小不能超过 5MB"     |
| 上传过程中网络中断    | 提示"网络异常，请重试"，不执行任何数据变更      |

### 6.3 数据格式异常

| 场景                | 处理方式                     |
| ----------------- | ------------------------ |
| 评分不是整数            | 校验报错："评分必须为0-100的整数"     |
| 评分超出范围            | 校验报错："评分必须为0-100的整数"     |
| URL格式不正确          | 校验报错："URL格式不正确"          |
| 是否队长填写非"是/否"      | 校验报错："是否队长只能选择是或否"       |
| 实力等级填写非 S/A/B/C/D | 校验报错："实力等级只能选择S/A/B/C/D" |
| 常用英雄名称不在英雄库中      | 校验报错："常用英雄'XXX'不在英雄库中"   |
| 直播间号包含非数字字符       | 校验报错："直播间号只需填写纯数字"       |
| 个人简介超过200字符       | 校验报错："个人简介不能超过200字符"     |
| 战队名称超长            | 校验报错："战队名称不能超过20个字符"     |
| 必填字段为空            | 校验报错："战队名称/位置不能为空"       |

### 6.4 并发处理

- 导入操作期间锁定战队数据，禁止同时进行其他战队管理操作
- 如果已有导入任务正在执行，提示"当前有导入任务正在进行，请稍后重试"

## 7. 技术方案要点

### 7.1 后端技术选型

| 技术              | 用途            | 说明                         |
| --------------- | ------------- | -------------------------- |
| xlsx / exceljs  | Excel 文件解析与生成 | 用于解析上传的 Excel 文件和生成模板/错误报告 |
| class-validator | 数据校验          | 复用现有 DTO 校验逻辑 + 自定义英雄库校验器  |
| multer          | 文件上传          | 复用现有上传中间件                  |

**英雄数据文件**：
- 文件路径：`/data/lol-champion-map.json`（部署时可挂载到服务器磁盘路径）
- 数据格式：
  ```json
  {
    "Aatrox": { "id": "Aatrox", "name": "暗裔剑魔", "title": "亚托克斯", "tags": ["Fighter"] },
    "Ahri": { "id": "Ahri", "name": "九尾妖狐", "title": "阿狸", "tags": ["Mage", "Assassin"] }
  }
  ```
- 字段映射：支持通过 `name`（中文名称）或 `title`（中文称号）两种方式匹配英雄

### 7.2 导入流程（后端）

1. 接收上传的 Excel 文件
2. 解析 Excel 文件，提取数据行
3. 按战队名称分组数据（同一战队的连续多行为一个战队）
4. 逐行/逐战队校验数据（格式校验 + 业务校验）
5. 若存在校验错误，收集所有错误并返回，不执行任何数据写入
6. 校验全部通过后，开启数据库事务
7. 对于每个战队：检查是否已存在 → 存在则覆盖，不存在则新增
8. 提交事务
9. 清除相关缓存
10. 返回导入结果

### 7.3 事务安全

- 导入操作使用数据库事务，确保原子性
- 任一战队的导入失败则整体回滚（校验阶段已排除错误，此处为运行时异常保障）
- 覆盖模式下：先删除原队员数据，再插入新队员数据，确保数据一致性
- 保留原战队ID，只更新战队名称、队标、宣言等字段

### 7.4 性能考虑

- Excel 文件限制最大 80 行数据（16支战队×5行），数据量小，无需分批处理
- 导入完成后清除战队相关缓存（`teams:all` 和 `team:{id}`）

### 7.5 文件处理逻辑

**复用现有上传逻辑**：
- 批量导入接口复用现有的文件上传中间件（multer）
- 配置文件上传目录（如 `/uploads/temp/`），限制上传文件大小（最大 5MB）

**临时文件管理**：
- 上传的 Excel 文件先保存到临时目录（如 `/uploads/temp/import-xxx.xlsx`）
- 导入完成后立即删除原文件，避免占用存储空间
- 若导入失败，也需清理临时文件

**流程示意**：
```
1. 接收上传的 Excel 文件 → 保存到临时目录
2. 解析 Excel 文件 → 校验数据
3. 若校验失败 → 返回错误，删除临时文件
4. 若校验成功 → 执行导入事务
5. 导入完成 → 删除临时文件
6. 返回导入结果
```
