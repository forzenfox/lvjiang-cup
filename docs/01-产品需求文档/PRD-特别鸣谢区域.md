## 1. 产品概览

### 1.1 项目背景

驴酱杯赛事得到了众多赞助方的大力支持，包括斗鱼官方、多位老板的资金赞助以及各种特色奖项赞助。同时，赛事的顺利举办离不开幕后工作人员的辛勤付出。目前网站主页面缺少对这些赞助方和工作人员的展示与感谢，需要新增一个特别鸣谢区域，以表达赛事组织方的诚挚谢意，同时增强赛事的社区氛围和赞助方的参与感。

### 1.2 目标用户

- **普通观众**：浏览网站时了解赛事赞助信息，感受赛事的社区氛围
- **赞助方**：在网站上看到自己的赞助展示，获得参与感和荣誉感
- **幕后工作人员**：获得公开认可和感谢，增强归属感
- **赛事组织方**：通过公开鸣谢维护与赞助方的良好关系

### 1.3 产品价值

- 增强赞助方的参与感和荣誉感，促进后续赛事的赞助合作
- 展示赛事的社区凝聚力，提升赛事品牌形象
- 对幕后工作人员的公开认可，激励更多志愿者参与
- 丰富主页内容，增加页面视觉趣味性和停留时长

### 1.4 业务目标

| 指标 | 当前值 | 目标值 | 测量周期 | 计算方式 | 目标依据 |
|------|--------|--------|----------|----------|----------|
| 主页用户平均停留时长 | 基准值 | +5% | 上线后 30 天 | 用户在主页停留总时长/访问次数 | 弹幕滚动内容吸引用户观看，参考同类网站弹幕模块停留增幅 |
| 赞助方满意度 | 无 | ≥80%正面反馈 | 上线后 30 天 | 赞助方反馈调查 | 赞助方获得公开展示后预期满意度提升 |
| 特别鸣谢区域曝光率 | 0 | ≥60% | 上线后 14 天 | 区域进入视口的访问次数/总访问次数 | 区域位于主页中下部，预估滚动到该区域的用户比例 |

### 2.2 详细功能描述

#### 2.2.1 弹幕滚动展示

- **展示方式**：水平弹幕，文字内容从右向左匀速移动
- **弹幕格式**：
  - 赞助方：`感谢老板{赞助人}赞助的{赞助内容}，老板大气`
  - 幕后工作人员：`感谢{工作人员}的辛勤付出`
- **滚动方向**：从右向左
- **滚动速度**：匀速移动，单条弹幕从完全进入到完全离开视口约 15 秒（PC 端）/ 12 秒（移动端）。滚动总时长根据内容总宽度动态计算：`scroll-duration = (contentWidth / viewportWidth) × baseTime`，其中 `baseTime` 为单屏宽度的滚动时间（PC 端 15 秒，移动端 12 秒）
- **排列方式**：所有鸣谢条目按顺序排列成一行，首尾相连循环滚动
- **间距**：相邻两条弹幕之间保持固定间距（约 80px）
- **循环**：所有弹幕条目滚动完毕后无缝循环，不出现空白间隔

#### 2.2.4 后台管理配置

**赞助信息管理**：

- 管理员可添加、编辑、删除赞助信息

### 2.4 业务规则

#### 2.4.1 正常流程规则

- 赞助信息添加后默认为启用状态（status=1）
- 排序规则：按 `order ASC` 排序；`order` 值相同时，按 `created_at ASC` 排序
- 前端仅展示 `status=1` 的赞助信息和工作人员
- 弹幕文本生成规则：`感谢老板{sponsor_name}赞助的{sponsor_content}，老板大气`
- 工作人员文本生成规则：按角色分组，格式为 `感谢{role}：{name1}、{name2}...的辛勤付出`

#### 2.4.2 边界规则

- **赞助人名称限制**：最多 30 个字符
- **赞助内容限制**：最多 50 个字符
- **特殊奖项说明限制**：最多 100 个字符
- **工作人员姓名限制**：最多 20 个字符
- **角色分类限制**：最多 20 个字符
- **弹幕条目数量**：无上限，所有启用状态的赞助信息均参与滚动
- **工作人员数量**：无上限

#### 2.4.3 异常处理

- **数据为空**：当无启用状态的赞助信息时，弹幕区域显示"感谢所有支持驴酱杯的朋友们"
- **数据为空（工作人员）**：当无启用状态的工作人员时，不展示工作人员区域
- **接口加载失败**：显示骨架屏，自动重试 3 次（间隔 1 秒），重试失败后显示"加载失败，请刷新重试"

## 3. 页面设计

### 3.1 设计风格

- **整体风格**：与网站现有暗黑主题保持一致
- **色彩方案**：
  - 区域背景：深色半透明背景（`rgba(0, 0, 0, 0.6)`），与网站整体风格协调
  - 弹幕文字：金色渐变（`#F59E0B` → `#FBBF24`），与网站主色调一致
  - 区域标题：白色大字 + 金色装饰线
  - 工作人员文字：浅灰色（`#D1D5DB`）
- **字体**：使用网站统一字体

### 3.2 页面布局

- **位置**：主页 ScheduleSection（赛程区域）下方
- **整体结构**：
  - 顶部：区域标题"特别鸣谢"
  - 中部：弹幕滚动区域（赞助信息）
  - 底部：幕后工作人员展示区域

### 3.3 交互设计

#### 3.3.1 弹幕滚动交互

- **自动滚动**：页面加载后弹幕自动开始从右向左滚动
- **悬停暂停**：PC 端鼠标悬停在弹幕区域时暂停滚动，移开后恢复
- **无缝循环**：所有弹幕条目首尾相连，滚动到末尾后无缝衔接开头
- **触摸交互**：移动端不提供暂停功能，持续滚动

#### 3.3.2 特殊奖项交互

- **PC 端**：直接展示所有特殊奖项列表
- **移动端**：默认收起，显示前 3 条 + "查看更多"按钮；点击展开显示全部；再次点击收起

#### 3.3.3 工作人员交互

- 静态展示，无特殊交互
- 按角色分组，每组一行

## 4. 技术实现

### 4.1 前端实现

- **技术栈**：使用现有网站前端技术栈（React + TypeScript + Tailwind CSS + Framer Motion）
- **弹幕组件**：基于 CSS `@keyframes` + `translateX` 实现水平匀速滚动动画
- **无缝循环**：复制一份弹幕内容拼接在末尾，当第一份完全滚出视口时重置位置
- **悬停暂停**：通过 `animation-play-state: paused` 实现

#### 4.1.1 弹幕滚动核心实现思路

```css
@keyframes marquee-scroll {
  0% {
    transform: translateX(0);
  }
  100% {
    transform: translateX(-50%);
  }
}

.marquee-container {
  overflow: hidden;
}

.marquee-content {
  display: flex;
  animation: marquee-scroll var(--scroll-duration) linear infinite;
}

.marquee-container:hover .marquee-content {
  animation-play-state: paused;
}
```

> **说明**：内容复制一份拼接，`translateX(-50%)` 使第一份完全移出时第二份刚好到达起始位置，实现无缝循环。`--scroll-duration` 根据内容总宽度动态计算。

### 4.5 性能优化方案

#### 4.5.1 弹幕滚动优化

- **CSS 动画优先**：使用 CSS `transform` + `animation` 实现滚动，利用 GPU 加速
- **避免重排重绘**：使用 `transform` 而非 `left`/`margin-left` 属性
- **will-change 提示**：为滚动容器添加 `will-change: transform`

#### 4.5.2 数据缓存

- **接口缓存**：赞助信息和工作人员数据缓存 10 分钟（localStorage）
- **静态数据**：初期数据量小，可考虑前端静态配置，后续数据增多后切换为接口获取

#### 4.5.3 渲染优化

- **视口检测**：仅当特别鸣谢区域进入视口时启动弹幕滚动动画
- **离开视口暂停**：区域离开视口时暂停动画，节省性能
