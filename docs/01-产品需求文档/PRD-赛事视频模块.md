# 主页面赛事视频模块 PRD

## 0. 术语表

| 术语        | 定义                                                |
| --------- | ------------------------------------------------- |
| bvid      | B 站视频的唯一标识符，格式为 `BV` 开头 + 10 位字符，如 `BV1xx411c7mD` |
| BV 号      | 同 bvid                                            |
| B 站嵌入式播放器 | B 站提供的可通过 iframe 嵌入第三方网站的网页视频播放器                  |
| 三屏布局      | PC 端视频轮播的展示形式：中间为主视频播放器，左右各展示一个相邻视频缩略图            |
| 单屏布局      | 移动端视频轮播的展示形式：仅展示当前主视频，无左右缩略图                      |
| 自动轮播      | 视频模块在无用户操作时，按设定时间间隔自动切换到下一个视频                     |
| 自动播放      | 视频加载后自动开始播放（受浏览器策略限制，可能需静音才能自动播放）                 |
| 静音自动播放    | 视频以静音状态自动播放，符合大多数浏览器的自动播放策略                       |

## 1. 产品概览

### 1.1 项目背景

随着赛事内容的不断丰富，用户对于赛事相关视频的需求日益增长。为了提升用户体验，增强网站的内容吸引力，需要在主页面新增一个视频模块，展示赛事相关的视频内容。

### 1.2 目标用户

- **普通观众**：对赛事感兴趣，希望快速了解赛事相关视频内容
- **赛事粉丝**：关注特定赛事，希望方便地观看赛事回放、精彩集锦等内容

### 1.3 产品价值

- 提升主页面的内容丰富度和视觉吸引力
- 为用户提供便捷的赛事视频观看入口
- 增加网站的用户停留时间和页面浏览量
- 满足运营团队对赛事内容推广的需求

### 1.4 业务目标

| 指标         | 当前值  | 目标值         | 测量周期     | 计算方式            | 目标依据                                                            |
| ---------- | ---- | ----------- | -------- | --------------- | --------------------------------------------------------------- |

## 2. 核心功能

### 2.1 功能概览

| 功能模块    | 功能描述                                                        | 优先级 |
| ------- | ----------------------------------------------------------- | --- |
| 视频轮播展示  | PC 端三屏布局：中间主视频播放器 + 左右缩略图预览；移动端单屏布局；支持手动切换和自动轮播；最多展示 10 个视频 | P0  |
| B 站视频嵌入 | 通过 B 站嵌入式播放器（iframe）播放视频内容                                  | P0  |
| 后台管理配置  | 管理员通过后台页面配置视频链接、标题、排序、状态等                                   | P0  |
| 响应式设计   | 支持 PC 端（三屏布局）、平板端（三屏布局缩小）、移动端（单屏布局）适配                       | P0  |

### 2.2 详细功能描述

#### 2.2.1 视频轮播展示

- **三屏布局（PC 端 / 平板端）**：中间展示主视频播放器，左右各展示一个相邻视频的缩略图和标题
- **单屏布局（移动端）**：仅展示当前主视频播放器，下方显示滚动指示器，支持左右滑动切换
- **自动轮播**：
  - PC 端：默认开启自动轮播，间隔 5 秒自动切换到下一个视频
  - 移动端：不开启自动轮播
  - 用户手动操作（点击箭头/缩略图/滑动）后，暂停自动轮播 30 秒，之后恢复
  - 到达最后一个视频后，循环回到第一个视频继续轮播
- **手动切换**：
  - PC 端：点击左右箭头按钮切换到上/下一个视频；点击左侧缩略图切换到上一个视频；点击右侧缩略图切换到下一个视频
  - 移动端：左右滑动切换到上/下一个视频（滑动距离 > 50px 触发切换，< 50px 回弹）
- **视频预览**：主视频显示完整 B 站播放器，左右缩略图显示视频封面图和标题
- **数量限制**：前端最多展示 10 个启用状态的视频；视频数量 ≤ 2 个时，隐藏左右箭头和缩略图区域

#### 2.2.2 B 站视频嵌入

**我方需开发的功能**：

- **嵌入式播放器集成**：使用 B 站提供的 iframe 嵌入式播放器，通过 `bvid` 和 `page` 参数加载指定视频
- **播放器参数配置**：通过 iframe URL 参数控制播放器初始行为（详见附录 10.1）
- **错误处理**：播放器加载失败时显示错误占位图和重试按钮
- **懒加载**：仅当前视频及左右相邻视频的 iframe 会被渲染

**B 站播放器自带功能（非我方开发，受 B 站策略影响）**：

- 播放/暂停、音量调节、全屏、倍速播放 — 由 B 站播放器提供
- 弹幕功能 — 由 B 站播放器提供，可通过 URL 参数 `danmaku=1/0` 控制初始状态
- 清晰度选择 — 由 B 站播放器提供，受用户 B 站登录状态影响，非我方可控
- 进度条、字幕等 — 由 B 站播放器提供

**已知限制**：

- B 站嵌入式播放器不支持我方自定义 UI 皮肤
- 弹幕开关仅控制初始状态，用户可在播放器内自行切换
- 清晰度受 B 站策略控制，未登录用户默认 360P，登录用户最高可选 1080P
- 部分视频因版权限制可能禁止嵌入第三方网站

#### 2.2.3 后台管理配置

**视频链接管理**（简化设计）：

- 管理员可添加、编辑、删除视频
- 添加/编辑时支持视频预览（在弹窗中预览 B 站播放器效果）
- **简化输入**：管理员只需粘贴 B 站视频链接，系统自动提取 bvid 和 page
- **自动获取元信息**：后端自动调用 B 站 API 获取视频标题和封面 URL
- **自定义标题**：管理员可输入自定义标题（可选），留空则使用 B 站原始标题
- **封面处理**：封面 URL 通过后端代理访问，解决跨域问题

**标题显示逻辑**：

- 当管理员设置了自定义标题时，显示自定义标题
- 自定义标题为空时，显示 B 站获取的视频原始标题

**封面处理**：

- B站封面图片（`i0.hdslb.com`、`i1.hdslb.com`、`i2.hdslb.com`）可直接在前端 `<img>` 标签使用
- `<img>`、CSS `background-image` 等HTML标签加载跨域图片不受CORS限制
- 只需在CSP中配置允许B站图片域名即可

**列表管理**：

- 视频列表支持分页展示（每页 20 条）
- 支持按标题搜索
- 支持按状态筛选（全部 / 启用 / 禁用）

**排序功能**：

- 支持拖拽排序（前端拖拽后提交视频 ID 顺序列表，后端自动计算并更新 order 值）

**状态管理**：

- 可启用/禁用单个视频（切换时需二次确认）
- 启用视频数量达到 10 个上限时，禁用"启用"操作并提示

**链接验证**：

- 输入时实时校验链接格式（前端校验）
- 提交时后端校验 bvid 有效性（调用 B 站 API 验证视频是否存在）
- 重复 bvid 检测（前端 + 后端双重校验）
- 操作日志：简单网站不需要操作日志记录

#### 2.2.4 响应式设计

- **PC 端（≥ 1024px）**：三屏布局，中间主视频 + 左右缩略图，支持点击箭头或缩略图切换
- **平板端（768px - 1023px）**：三屏布局缩小版，左右缩略图尺寸缩小，交互与 PC 端一致
- **移动端（< 768px）**：单屏布局，仅展示主视频，支持左右滑动切换
- **适配范围**：移动端适配仅针对网站主页面，后台管理页面仅适配 PC 端（移动端访问后台时显示"请使用电脑访问"提示页）

### 2.3 数据结构定义

#### 2.3.1 视频数据表结构

| 字段名           | 类型       | 必填 | 默认值  | 说明                           | 示例                                     |
| --------------- | -------- | -- | ---- | ------------------------------ | -------------------------------------- |
| id              | string   | 是  | 自动生成 | 视频唯一标识（UUID）                 | "a1b2c3d4-e5f6-7890-abcd-ef1234567890" |
| bvid            | string   | 是  | -    | B 站视频 ID（唯一索引）               | "BV1xx411c7mD"                         |
| page            | number   | 否  | 1    | B 站视频分P页码（从1开始）              | 1                                      |
| bilibili\_title | string   | 否  | null | B 站原始标题（API获取）               | "2026 驴酱杯决赛集锦"                         |
| custom\_title   | string   | 否  | null | 自定义标题（可选）                    | "驴酱杯精彩集锦"                             |
| cover\_url      | string   | 否  | null | B 站封面图 URL（API获取）             | "https://i1.hdslb.com/bfs/archive/xxx.jpg" |
| order           | number   | 是  | 0    | 排序值，越小越靠前                    | 1                                      |
| status          | enum     | 是  | 1    | 状态：0-禁用，1-启用                 | 1                                      |
| created\_at     | datetime | 是  | 自动填充 | 创建时间                         | 2025-01-15 10:00:00                    |
| updated\_at     | datetime | 是  | 自动填充 | 更新时间                         | 2025-01-15 10:00:00                    |
| created\_by     | string   | 是  | -    | 创建人（管理员 ID）                  | "admin\_001"                           |

> **说明**：
> - `bilibili_title`：从 B 站 API 获取的原始标题，作为备用
> - `custom_title`：管理员自定义标题，优先级高于 `bilibili_title`
> - 前端显示标题逻辑：`custom_title || bilibili_title || '未命名视频'`

#### 2.3.2 前端展示数据结构

```typescript
interface VideoItem {
  id: string;
  title: string;        // 显示标题（已处理优先级：custom_title > bilibili_title）
  bvid: string;
  page: number;
  coverUrl?: string;    // 封面图片 URL（已转换为代理地址）
}

interface VideoListResponse {
  code: number;
  data: {
    list: VideoItem[];
    total: number;
  };
  message?: string;
}
```

> **说明**：前端展示数据结构不包含 `order` 和 `status` 字段。排序由后端按 `order ASC, created_at DESC` 处理后返回；前端仅展示 `status=1` 的视频。

### 2.4 业务规则

#### 2.4.1 正常流程规则

- 视频添加后默认为启用状态（status=1）
- 排序规则：按 `order ASC` 排序；`order` 值相同时，按 `created_at DESC` 排序
- 前端最多展示 10 个启用状态的视频
- 后台可存储超过 10 个视频，但前端仅展示 order 最小的 10 个启用视频
- 视频切换后，新视频按自动播放策略处理（详见 §2.2.1）

#### 2.4.2 边界规则

- **链接格式校验**：
  - 支持的输入格式：
    - 直接输入 bvid：`BV1xx411c7mD`
    - B 站完整链接：`https://www.bilibili.com/video/BV1xx411c7mD`
    - B 站带分P参数链接：`https://www.bilibili.com/video/BV1xx411c7mD?p=2`
    - B 站带追踪参数链接：`https://www.bilibili.com/video/BV1xx411c7mD/?spm_id_from=xxx`
  - bvid 提取规则：
    - 完整链接：使用正则 `/BV[a-zA-Z0-9]+/` 从 URL 中提取
    - 分P参数：使用正则 `/[?&]p=(\d+)/` 提取 page 值，默认为 1
  - 前端校验：输入时实时校验格式，不符合时显示"请输入有效的 B 站视频链接或 BV 号"
  - 后端校验：提交时验证 bvid 对应的视频在 B 站是否存在
  - **不支持短链接**：不支持 b23.tv 短链接格式，如用户输入短链接，提示"请使用完整链接或直接输入 BV 号"
- **标题限制**：最多 50 个字符（包含中文、英文、数字、标点）
- **重复检测**：同一 bvid + page 组合不可重复添加
- **数量限制**：最多启用 10 个视频，超出时提示"最多可启用 10 个视频，请先禁用部分视频"
- **排序范围**：order 值范围 0-9999

#### 2.4.3 异常处理

- **B 站视频失效**：
  - 前端检测：iframe 加载超时（10 秒无响应）或 B 站返回错误页面时，显示"视频已下架或不存在"占位图
  - 后台标记：后台视频列表中显示"异常"标签，发送站内通知给管理员
- **网络加载失败**：
  - 自动重试 3 次（间隔 1 秒）
  - 重试失败后显示"网络连接异常，请刷新重试" + 刷新按钮
- **超出数量限制**：
  - 后台提示"最多可启用 10 个视频，请先禁用部分视频"
  - 不允许继续启用新视频
- **无效链接**：
  - 提交时即时校验，显示"链接格式不正确，请输入有效的 B 站视频链接"
  - 不允许保存
- **B 站播放器嵌入被拒绝**：
  - 部分视频因版权限制禁止嵌入，显示"该视频暂不支持站外播放"占位图
  - 后台标记为"嵌入受限"，通知管理员

## 3. 页面设计

### 3.1 设计风格

- **整体风格**：参考 LPL 官方页面设计，保持与现有网站风格一致
- **色彩方案**：使用网站主色调，确保视觉一致性
- **字体**：使用网站统一字体，保证品牌识别度

### 3.2 页面布局

- **位置**：主页面核心位置，用户进入网站后一眼可见
- **尺寸**：宽度与主页面一致，高度适中，确保不影响其他内容展示
- **布局**：
  - 顶部：模块标题（如"赛事精彩视频"）
  - 中部：视频轮播区域
    - **PC 端 / 平板端**：三屏布局
      - 左侧：上一个视频的缩略图预览（含封面图 + 标题）
      - 中间：当前主视频播放器
      - 右侧：下一个视频的缩略图预览（含封面图 + 标题）
      - 左右两侧边缘：半透明箭头按钮
    - **移动端**：单屏布局
      - 中间：当前主视频播放器
      - 底部：滚动指示器（圆点样式，当前视频高亮）
  - 底部：查看更多链接（可选）

### 3.3 交互设计

#### 3.3.1 视频切换交互

- **PC 端 / 平板端**：
  - 点击左箭头 → 切换到上一个视频
  - 点击右箭头 → 切换到下一个视频
  - 点击左侧缩略图 → 切换到上一个视频
  - 点击右侧缩略图 → 切换到下一个视频
  - 到达边界（第一个/最后一个）时，循环切换
- **移动端**：
  - 向左滑动（距离 > 50px）→ 切换到下一个视频
  - 向右滑动（距离 > 50px）→ 切换到上一个视频
  - 滑动距离 ≤ 50px → 回弹到当前视频
  - 到达边界时，循环切换

#### 3.3.2 播放交互

- **PC 端**：
  - 首次加载：主视频静音自动播放（符合浏览器自动播放策略）
  - 切换视频：新视频静音自动播放
  - 用户可手动取消静音
- **移动端**：
  - 首次加载：不自动播放，显示封面图 + 居中播放按钮
  - 切换视频：不自动播放，显示新视频封面图 + 播放按钮
  - 用户点击播放按钮后开始播放

#### 3.3.3 自动轮播交互

- **触发条件**：PC 端默认开启，移动端不开启
- **轮播间隔**：5 秒
- **暂停规则**：用户手动操作（点击箭头/缩略图/滑动/播放器内操作）后，暂停自动轮播 30 秒
- **恢复规则**：暂停 30 秒后自动恢复轮播
- **循环规则**：到达最后一个视频后，循环回到第一个视频

#### 3.3.4 悬停效果

- PC 端鼠标悬停在左右缩略图上时，缩略图放大 1.05 倍（200ms ease-in-out）
- 鼠标悬停超过 200ms 时，预加载对应视频的 iframe

### 3.4 视觉元素

- **主视频播放器**：中间位置，尺寸最大，显示完整的 B 站播放器
- **左右缩略图**：
  - 位置：主视频的左右两侧
  - 尺寸：主视频宽度的 40%，高度按 16:9 等比缩放
  - 效果：半透明遮罩（opacity: 0.7），悬停时取消遮罩
  - 内容：显示视频封面图和标题（标题最多显示 2 行，超出省略号）
- **滚动指示器**：
  - PC 端：显示"当前序号/总数"（如 3/10）
  - 移动端：圆点指示器，当前视频高亮
- **控制按钮**：左右半透明箭头按钮，样式与整体设计风格一致

### 3.5 状态设计

#### 3.5.1 加载状态

- **首屏加载**：显示骨架屏（灰色占位块，模拟播放器和缩略图区域）
- **视频切换**：显示 Loading 旋转动画（时长 300ms）

#### 3.5.3 错误状态

## 4. 技术实现

### 4.1 前端实现

- **技术栈**：使用现有网站的前端技术栈
- **轮播组件**：实现左右滚动的轮播组件，支持触摸滑动和键盘导航
- **B 站播放器集成**：使用 B 站提供的 iframe 嵌入式播放器
  - 嵌入方式：使用 iframe 标签嵌入 B 站播放器
  - 参数配置：根据场景配置播放器参数（详见附录 10.1）

### 4.2 后端实现

- **视频管理 API**：提供视频链接的增删改查接口
- **数据存储**：存储视频链接、标题、排序等信息
- **链接验证**：
  - bvid 格式校验（正则匹配）
  - 视频有效性验证（调用 B 站 API 检查视频是否存在）
- **bvid 唯一索引**：数据库对 bvid + page 建立唯一索引，防止重复

### 4.3 后台管理

- **管理界面**：在现有后台系统中添加视频管理模块
- **操作权限**：只有管理员角色可操作视频管理功能
- **操作流程**：
  1. 管理员登录后台
  2. 进入视频管理模块
  3. 执行管理操作：
     - 添加视频：输入标题 + bvid/链接 → 实时校验 → 预览确认 → 保存
     - 编辑视频：修改标题/链接 → 预览确认 → 保存
     - 删除视频：二次确认 → 删除
     - 调整排序：拖拽排序 → 保存
     - 启用/禁用：二次确认 → 切换状态
  4. 查看视频列表

  ```json
          "page": 1,
          "coverUrl": "https://example.com/cover.jpg"
        }
      ],
      "total": 10
    },
    "message": "success"
  }
  ```

### 4.5 性能优化方案

#### 4.5.1 懒加载策略

- **首屏加载**：仅加载当前视频及左右相邻视频（共 3 个）
- **视口加载**：其他视频进入视口前 500px 时开始加载
- **iframe 懒加载**：使用 `loading="lazy"` 属性

#### 4.5.2 预加载策略

- **悬停预加载**：鼠标悬停缩略图超过 200ms 时，预加载对应视频 iframe
- **空闲预加载**：使用 `requestIdleCallback` 在浏览器空闲时预加载

#### 4.5.3 防抖节流

- **切换防抖**：视频切换操作防抖 300ms，避免频繁切换
- **滚动节流**：滚动事件节流 100ms

#### 4.5.4 缓存策略

- **接口缓存**：视频列表接口数据缓存 5 分钟（localStorage）
- **播放器缓存**：使用 B 站播放器的内置缓存
- **图片缓存**：封面图设置长期缓存（Cache-Control: max-age=31536000）

#### 4.5.5 渲染优化

- **限制同时渲染**：最多同时渲染 3 个 iframe（当前 + 左右）
- **离屏销毁**：超出可视范围的 iframe 从 DOM 移除
- **使用 CSS transform**：动画使用 transform 而非 left/top 属性

- **流量优化**：移动端默认加载 360P 清晰度（由 B 站播放器根据网络自动适配）

- 采用三屏布局缩小版，缩略图宽度为主视频的 35%

### 7.2 回滚方案

#### 7.2.1 快速回滚

- **功能开关**：通过配置中心控制视频模块开关，出现问题时一键关闭
- **灰度发布**：先对 10% 用户开放，验证无误后全量发布
- **监控告警**：设置错误率告警阈值（>1% 触发告警）

#### 7.2.2 代码回滚

- **版本保留**：保留上一个稳定版本 tag，10 分钟内完成回滚
- **数据库兼容**：数据库变更需支持向下兼容，可快速回退
- **回滚流程**：
  1. 发现问题后立即关闭功能开关
  2. 评估问题严重程度
  3. 如需回滚，执行回滚脚本
  4. 验证回滚后功能正常
  5. 问题修复后重新发布

#### 7.2.3 数据回滚

- **备份策略**：每日备份视频数据表
- **回滚脚本**：准备数据回滚 SQL 脚本
- **验证流程**：回滚后验证数据完整性

## 9. 附录

### 9.1 B 站嵌入播放器

#### 9.1.1 iframe 嵌入示例

```html
<iframe
  src="//player.bilibili.com/player.html?bvid=BV1xx411c7mD&page=1&danmaku=1&autoplay=1&muted=1&high_quality=1"
  scrolling="no"
  border="0"
  frameborder="no"
  framespacing="0"
  allowfullscreen="true"
  width="100%"
  height="100%">
</iframe>
```

#### 9.1.2 播放器参数说明

| 参数            | 类型     | 默认值 | 说明                                    |
| ------------- | ------ | --- | ------------------------------------- |
| bvid          | string | -   | B 站视频 ID（必填）                          |
| page          | number | 1   | 分P视频页码                                |
| danmaku       | 0/1    | 1   | 弹幕开关：1-开启，0-关闭                        |
| autoplay      | 0/1    | 0   | 自动播放：1-开启，0-关闭                        |
| muted         | 0/1    | 0   | 静音：1-静音，0-非静音（autoplay=1 时建议 muted=1） |
| high\_quality | 0/1    | 0   | 高画质：1-优先高画质（需登录）                      |

#### 9.1.3 各场景参数配置

| 场景       | autoplay | muted | 说明                 |
| -------- | -------- | ----- | ------------------ |
| PC 端首次加载 | 1        | 1     | 静音自动播放，符合浏览器策略     |
| PC 端切换视频 | 1        | 1     | 静音自动播放新视频          |
| 移动端首次加载  | 0        | 0     | 不自动播放，显示封面图 + 播放按钮 |
| 移动端切换视频  | 0        | 0     | 不自动播放，显示封面图 + 播放按钮 |

> **注意**：移动端设置 `autoplay=1` 也可能因浏览器策略被阻止，因此移动端统一使用 `autoplay=0`，由用户手动触发播放。

### 9.2 B 站播放器已知限制

- 嵌入式播放器不支持自定义 UI 皮肤
- 部分视频因版权限制禁止嵌入第三方网站（B 站返回 403）
- 清晰度受 B 站策略控制：未登录默认 360P，登录后最高 1080P
- 弹幕内容由 B 站管理，我方无法控制弹幕内容
- 播放器加载速度受 B 站 CDN 影响，我方无法优化
- iframe 跨域限制，我方无法通过 JS 控制 B 站播放器的播放/暂停等操作

### 9.3 CSP 安全策略配置

为允许 B 站播放器正常加载，需在 CSP 中添加以下规则：

```
Content-Security-Policy:
  frame-src https://player.bilibili.com;
  img-src https://i0.hdslb.com https://i1.hdslb.com https://i2.hdslb.com;
  connect-src https://api.bilibili.com;
```

### 9.5 FAQ

| 问题                       | 回答                                                                                               |
| ------------------------ | ------------------------------------------------------------------------------------------------ |
| 为什么 PC 端静音自动播放而不是有声自动播放？ | 主流浏览器（Chrome 66+、Safari 11+、Firefox 66+）的自动播放策略要求：有声视频必须由用户手势触发才能播放，静音视频可自动播放。因此 PC 端采用静音自动播放策略。 |
| 为什么移动端不支持自动播放？           | iOS Safari 严格禁止自动播放（即使静音），Android Chrome 行为不一致。为保障一致体验，移动端统一不自动播放。                               |
| B 站视频禁止嵌入怎么办？            | 后台添加视频时会校验嵌入权限，如视频禁止嵌入则提示管理员。前端如遇到嵌入失败，显示"该视频暂不支持站外播放"占位图。                                       |
| 视频数量少于 3 个时怎么展示？         | 视频数量 ≤ 2 个时，隐藏左右箭头和缩略图区域，仅展示主视频播放器。                                                              |
| 自动轮播会影响用户观看体验吗？          | 自动轮播仅切换展示的视频，不会打断用户正在观看的视频。用户操作后暂停轮播 30 秒，给予充分的观看时间。                                             |
