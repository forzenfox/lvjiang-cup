## 三、页面一：对战信息详情页（变更）

### 3.1 变更说明

在现有的 `MatchDetailModal` 弹框中添加"对战数据"按钮，仅在管理员上传了对战数据后显示。

### 3.2 按钮设计规范

#### 按钮位置

位于弹框底部操作区域，与现有内容保持间距。

```
┌──────────────────────────────────────┐
│  [关闭 X]                            │
│                                      │
│  对战时间: 2026-04-16 14:00          │
│  ┌──────────────────────────────┐    │
│  │  [队A Logo]  2 : 0  [队B Logo] │  │
│  │   BLG              WBG       │    │
│  └──────────────────────────────┘    │
│                                      │
│  队员对阵表                           │
│  [选手A] VS [选手B]                  │
│  [选手A] VS [选手B]                  │
│  ...                                 │
│                                      │
│  BO3                                 │
│                                      │
│  ┌──────────────────────────────┐    │
│  │  📊 对战数据                   │ ← 新增按钮
│  └──────────────────────────────┘    │
└──────────────────────────────────────┘
```

#### 按钮样式

| 属性 | 值 | Tailwind类名 |
|------|-----|-------------|
| 宽度 | 100%（满宽） | `w-full` |
| 高度 | 44px | `h-11` |
| 背景色 | 金色渐变 | `bg-gradient-to-r from-yellow-400 to-yellow-600` |
| 文字色 | 深蓝 | `text-[#1E3A8A]` |
| 字号 | 16px | `text-base` |
| 字重 | 700 | `font-bold` |
| 圆角 | 8px | `rounded-lg` |
| 图标 | BarChart3 (lucide-react) | `<BarChart3 className="w-5 h-5" />` |
| 内边距 | 0 16px | `px-4` |
| 间距 | margin-top 16px | `mt-4` |
| 光晕 | 金色阴影 | `hover:shadow-[0_0_15px_rgba(250,204,21,0.5)]` |

#### 按钮状态

| 状态 | 样式 |
|------|------|
| 默认 | `bg-gradient-to-r from-yellow-400 to-yellow-600 text-[#1E3A8A]` |
| 悬停 | `hover:shadow-[0_0_15px_rgba(250,204,21,0.5)] hover:brightness-110 cursor-pointer` |
| 点击 | `active:scale-[0.98]` |
| 禁用 | `bg-gray-600 text-gray-400 cursor-not-allowed opacity-50` |
| 隐藏 | 不渲染DOM（未上传数据时不显示，非禁用） |

#### 交互说明

- 点击后在新标签页打开对战数据详情页：`/match/{id}/games`
- 鼠标悬停显示手型指针 `cursor-pointer`
- 悬停时轻微放大效果 `hover:brightness-110` + 金色光晕

### 3.3 组件代码参考

```tsx
{/* MatchDetailModal 底部新增 */}
{hasMatchData && (
  <button
    onClick={() => window.open(`/match/${match.id}/games`, '_blank')}
    className="w-full h-11 mt-4 flex items-center justify-center gap-2
               bg-gradient-to-r from-yellow-400 to-yellow-600
               text-[#1E3A8A] font-bold text-base rounded-lg
               hover:shadow-[0_0_15px_rgba(250,204,21,0.5)]
               hover:brightness-110 active:scale-[0.98]
               transition-all duration-200 cursor-pointer"
  >
    <BarChart3 className="w-5 h-5" />
    对战数据
  </button>
)}
```

### 7.5 数据归一化规则

```
normalizedValue = Math.max(0, value) / Math.max(leftValue, rightValue, 1)
```

- 百分比类型（伤害占比、参团率、伤转）：直接使用百分比值 / 100
- 数值类型（分均补刀、分均经济等）：按双方最大值归一化
- 生存维度（可为负值）：负值映射为0，正值为 (value / maxValue)

### 7.7 技术实现建议

**推荐方案**：SVG绘制

- 使用 `<polygon>` 绘制网格和数据多边形
- 使用 `<text>` 绘制维度标签
- 使用 CSS transition 实现展开/收起动画
- 优势：矢量无损缩放、DOM可访问性好、易于添加交互事件
- 推荐库：Recharts（RadarChart组件）或自定义SVG
