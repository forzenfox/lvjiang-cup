# 战队信息扩展 UI 设计文档

## 1. 设计概述

### 1.1 设计目标
为"绿江杯"英雄联盟赛事网站的战队信息扩展功能提供统一的 UI/UX 设计规范，确保视觉一致性、交互流畅性和用户体验的连贯性。

### 1.2 设计原则
- **电竞风格**：深色主题 + 金色点缀，营造竞技氛围
- **信息层次**：清晰的信息架构，重要信息突出展示
- **交互友好**：直观的操作反馈，降低用户认知成本
- **响应式适配**：支持桌面端、平板端、移动端多设备展示

### 1.3 设计范围
- 战队信息卡片（前端展示）
- 队员详情弹框
- 管理后台配置页面
- 英雄选择器组件

## 4. 交互设计

### 4.1 战队卡片交互

#### 4.1.1 悬停效果
- **触发**：鼠标悬停在卡片上
- **动画**：
  - 卡片上移 `translateY(-8px)`
  - 边框颜色变为金色半透明
  - 阴影增强并添加金色光晕
- **时长**：`300ms`
- **缓动**：`cubic-bezier(0.4, 0, 0.2, 1)`

#### 4.1.2 队员区域点击
- **触发**：点击"队员：X 人"区域
- **反馈**：
  - 文字颜色变为金色
  - 显示下划线
- **动作**：打开队员详情弹框
- **过渡**：弹框从底部滑入 + 淡入

### 4.2 队员详情弹框交互

#### 4.2.1 打开动画
- **初始状态**：`opacity: 0, transform: translateY(20px) scale(0.95)`
- **结束状态**：`opacity: 1, transform: translateY(0) scale(1)`
- **时长**：`300ms`
- **缓动**：`cubic-bezier(0.4, 0, 0.2, 1)`

#### 4.2.2 关闭动画
- **触发**：点击关闭按钮或遮罩层
- **动画**：打开动画的反向
- **时长**：`200ms`

#### 4.2.3 英雄名称悬停
- **触发**：鼠标悬停在英雄名称上
- **反馈**：
  - 文字颜色变为金色
  - 显示英雄头像浮层
- **浮层动画**：`opacity 0→1, translateY(4px)→0`
- **时长**：`200ms`

#### 4.2.4 直播间链接点击
- **触发**：点击直播间链接
- **反馈**：
  - 文字添加下划线
  - 颜色加深
- **动作**：在新标签页打开斗鱼直播页面

### 4.3 表单交互

#### 4.3.1 输入框聚焦
- **触发**：点击输入框
- **反馈**：
  - 边框颜色变为蓝色 `#3B82F6`
  - 添加蓝色阴影 `0 0 0 3px rgba(59, 130, 246, 0.2)`
- **时长**：`150ms`

#### 4.3.2 字符计数
- **触发**：在文本域输入文字
- **反馈**：
  - 实时更新计数器
  - 超过 90% 时变为橙色
  - 超过 100% 时变为红色并禁止继续输入

#### 4.3.3 保存按钮
- **触发**：点击保存按钮
- **状态**：
  - 加载中：显示旋转图标 + "保存中..."
  - 成功：显示对勾图标 + "保存成功"（绿色提示）
  - 失败：显示错误图标 + 错误信息（红色提示）

### 4.4 英雄选择器交互

#### 4.4.1 搜索过滤
- **触发**：在搜索框输入文字
- **反馈**：实时过滤英雄列表
- **动画**：列表项淡入淡出
- **时长**：`150ms`

#### 4.4.2 英雄选择
- **触发**：点击英雄项
- **反馈**：
  - 添加选中边框和背景
  - 显示对勾图标
  - 更新已选计数
- **动画**：缩放脉冲 `scale(1.02)`
- **时长**：`150ms`

#### 4.4.3 达到上限
- **触发**：已选 5 个英雄后点击其他英雄
- **反馈**：
  - 显示提示"最多选择 5 个英雄"
  - 提示自动消失
- **提示样式**：红色文字，底部弹出

## 6. 动画规范

### 6.1 过渡时间
| 类型 | 时长 | 用途 |
|-----|------|------|
| 快速 | `150ms` | 按钮悬停、输入框聚焦 |
| 标准 | `200ms` | 颜色变化、透明度变化 |
| 中等 | `300ms` | 卡片悬停、弹框打开 |
| 慢速 | `500ms` | 页面过渡、复杂动画 |

### 6.2 缓动函数
| 名称 | 值 | 用途 |
|-----|-----|------|
| 默认 | `ease` | 一般过渡 |
| 平滑 | `cubic-bezier(0.4, 0, 0.2, 1)` | 卡片悬停、弹框 |
| 弹跳 | `cubic-bezier(0.68, -0.55, 0.265, 1.55)` | 提示、徽章 |
| 线性 | `linear` | 旋转、进度条 |

### 6.3 关键帧动画

```css
/* 脉冲效果 */
@keyframes pulse {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.7; }
}

/* 闪烁效果 */
@keyframes shimmer {
  0% { background-position: -200% 0; }
  100% { background-position: 200% 0; }
}

/* 弹跳效果 */
@keyframes bounce {
  0%, 100% { transform: translateY(0); }
  50% { transform: translateY(-4px); }
}

/* 旋转效果 */
@keyframes spin {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}
```

## 8. 可访问性规范

### 8.1 颜色对比度
- 文字与背景对比度 ≥ 4.5:1
- 大文字（≥18px）对比度 ≥ 3:1
- 交互元素对比度 ≥ 3:1

### 8.2 焦点状态
- 所有可交互元素必须有可见的焦点指示器
- 焦点样式：`outline: 2px solid #3B82F6, outline-offset: 2px`
- 焦点顺序遵循逻辑阅读顺序

### 8.3 键盘导航
- 所有功能可通过键盘访问
- Tab 键在表单元素间切换
- Enter/Space 激活按钮和链接
- Esc 关闭弹框和菜单

### 8.4 屏幕阅读器
- 所有图片必须有 alt 文本
- 表单元素必须有 label 关联
- 动态内容更新需要 aria-live 通知
- 弹框需要 aria-modal 和 aria-labelledby
