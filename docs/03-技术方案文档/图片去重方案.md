# 图片去重方案

## 目标

对所有上传的图片（队标、队员头像、主播海报、视频封面）实现基于 MD5 的去重处理，避免同一图片重复上传浪费存储空间。

## 影响范围

- **后端**：`database.service.ts`、`upload.service.ts`、`videos.service.ts`
- **数据库**：新增 `file_hashes` 表
- **前端**：无影响

---

## 方案选型

### 推荐方案：数据库去重表 + MD5 哈希

**原理**：
```
上传文件 → 计算MD5 → 查询file_hashes表
                          ↓
              存在 → 返回已有文件路径
              不存在 → 保存文件 → 记录哈希 → 返回新路径
```

**优势**：
- 同一图片只计算一次 MD5，永久缓存
- 跨类型复用（不同模块用相同图片也能识别）
- 可查询统计重复文件
- 兼容现有代码结构

### 阶段一：数据库变更

#### 步骤 1.1：创建 file_hashes 表

**文件**: `backend/src/database/database.service.ts`

在 `initializeDatabase` 方法中添加表创建语句：

```sql
CREATE TABLE IF NOT EXISTS file_hashes (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  hash TEXT NOT NULL UNIQUE,
  file_path TEXT NOT NULL,
  file_type TEXT NOT NULL CHECK(file_type IN ('avatar', 'logo', 'poster', 'cover')),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE UNIQUE INDEX IF NOT EXISTS idx_file_hash ON file_hashes(hash);
```

#### 步骤 1.2：添加数据库操作方法

**文件**: `backend/src/database/database.service.ts`

添加以下方法：

```typescript
/**
 * 根据哈希查找已存在的文件
 */
async findFileByHash(hash: string): Promise<{ file_path: string } | undefined> {
  return this.get<{ file_path: string }>(
    'SELECT file_path FROM file_hashes WHERE hash = ?',
    [hash]
  );
}

/**
 * 记录文件哈希
 */
async recordFileHash(hash: string, filePath: string, fileType: FileType): Promise<void> {
  await this.run(
    'INSERT INTO file_hashes (hash, file_path, file_type) VALUES (?, ?, ?)',
    [hash, filePath, fileType]
  );
}

/**
 * 删除文件哈希记录
 */
async deleteFileHash(hash: string): Promise<void> {
  await this.run('DELETE FROM file_hashes WHERE hash = ?', [hash]);
}
```

### 阶段二：UploadService 重构

#### 步骤 2.1：添加类型定义和哈希计算方法

**文件**: `backend/src/modules/upload/upload.service.ts`

添加 `FileType` 类型和 MD5 计算方法：

```typescript
type FileType = 'avatar' | 'logo' | 'poster';

async computeFileHash(buffer: Buffer): Promise<string> {
  const crypto = require('crypto');
  return crypto.createHash('md5').update(buffer).digest('hex');
}
```

#### 步骤 2.2：重构上传方法

修改现有的上传方法，添加去重逻辑：

```typescript
async uploadTeamLogo(filename: string, buffer: Buffer): Promise<UploadResult> {
  const hash = await this.computeFileHash(buffer);

  // 检查是否已存在相同哈希的文件
  const existing = await this.databaseService.findFileByHash(hash);
  if (existing) {
    this.logger.log(`Logo file already exists, reusing: ${existing.file_path}`);
    return { url: getTeamLogoUrl(path.basename(existing.file_path)) };
  }

  // 保存新文件
  const logoPath = getTeamLogoPath(filename);
  await this.ensureDir(path.dirname(logoPath));
  await fs.promises.writeFile(logoPath, buffer);

  // 记录哈希
  await this.databaseService.recordFileHash(hash, logoPath, 'logo');

  this.logger.log(`Team logo uploaded: ${filename} (hash: ${hash})`);
  return { url: getTeamLogoUrl(filename) };
}

// uploadMemberAvatar 和 uploadStreamerPoster 类似处理
```

#### 步骤 2.3：更新文件删除逻辑

当删除文件时，同时删除哈希记录：

```typescript
async deleteFile(filePath: string, fileType: FileType): Promise<void> {
```

### 阶段三：VideosService 集成

#### 步骤 3.1：重构 fetchAndSaveCover

**文件**: `backend/src/modules/videos/videos.service.ts`

将封面下载纳入统一的去重体系：

```typescript
async fetchAndSaveCover(bvid: string): Promise<string> {
  const meta = await this.fetchBilibiliMeta(bvid);

  await this.ensureCoverDir();

  // 下载封面到临时缓冲区
  const coverBuffer = await this.downloadImageToBuffer(meta.coverUrl);
  const hash = await this.computeFileHash(coverBuffer);

  // 检查是否已存在相同哈希
  const existing = await this.databaseService.findFileByHash(hash);
  if (existing) {
    this.logger.log(`Cover already exists for ${bvid}, reusing: ${existing.file_path}`);
    return path.basename(existing.file_path);
  }

  // 生成新文件名并保存
  const filename = `${crypto.randomUUID()}.jpg`;
  const coverPath = getVideoCoverPath(filename);
  await fs.promises.writeFile(coverPath, coverBuffer);

  // 记录哈希
  await this.databaseService.recordFileHash(hash, coverPath, 'cover');

  this.logger.log(`Cover downloaded for ${bvid}: ${filename}`);
  return filename;
}

private async downloadImageToBuffer(url: string): Promise<Buffer> {
  return new Promise((resolve, reject) => {
    const protocol = url.startsWith('https') ? https : http;
    const chunks: Buffer[] = [];

    protocol.get(url, {
      headers: {
        'Referer': 'https://www.bilibili.com/',
        'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36',
      },
    }, (response) => {
      if (response.statusCode !== 200) {
        reject(new Error(`Failed to download: ${response.statusCode}`));
        return;
      }
      response.on('data', (chunk: Buffer) => chunks.push(chunk));
      response.on('end', () => resolve(Buffer.concat(chunks)));
      response.on('error', reject);
    }).on('error', reject);
  });
}
```

### 阶段四：孤立文件清理更新

#### 步骤 4.1：更新清理逻辑

**文件**: `backend/src/modules/upload/upload.service.ts`

修改 `cleanupOrphanedFiles` 方法，同时清理哈希表：

```typescript
async cleanupOrphanedFiles(): Promise<CleanupResult> {
  // ... 现有逻辑 ...

  // 清理孤立的哈希记录（文件已删除但哈希记录还在）
  const allHashes = await this.databaseService.all<{ hash: string; file_path: string }>(
    'SELECT hash, file_path FROM file_hashes'
  );

  for (const record of allHashes) {
    if (!fs.existsSync(record.file_path)) {
      await this.databaseService.deleteFileHash(record.hash);
      result.deletedFiles.push(record.file_path);
      this.logger.log(`Cleaned up orphaned hash record: ${record.hash}`);
    }
  }

  return result;
}
```

```
         ↓

└────────┬────────┘      └──────┬───────┘
         │                      │
         │ 否                   │ 是
         ↓                      ↓
```

## 测试要点

1. **同一图片上传两次**：第二次应复用第一次的文件和路径
2. **不同图片上传**：应各自保存
3. **删除后重新上传**：应重新保存新文件
4. **不同类型用相同图片**：应复用（如队员头像和队标用同一图片）

---

## 风险与回滚

### 风险
- 并发上传相同图片可能导致重复写入（可通过数据库 UNIQUE 约束防止）

### 回滚方案
- 删除 file_hashes 表和相关代码即可
- 已保存的文件不受影响
