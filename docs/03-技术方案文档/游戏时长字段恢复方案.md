# 游戏时长字段恢复方案

> **文档版本**：v1.0
> **创建日期**：2026-04-25
> **问题来源**：游戏时长字段被移除后，雷达图维度计算受到影响
> **影响范围**：雷达图维度计算、导入模板、数据库、前端展示

---

## 一、问题分析

### 1.1 游戏时长字段的影响链路

```
Excel 导入模板 (游戏时长)
    ↓
后端解析 (match-excel.util.ts)
    ↓
数据库存储 (match_games.game_duration)
    ↓
API 返回 (gameData.gameDuration)
    ↓
前端组件 (RadarChart.tsx)
    ↓
雷达图维度计算 (radarCalculations.ts)
```

### 1.2 受影响的雷达图维度计算

根据 `radarCalculations.ts` 分析，以下维度计算**依赖游戏时长**：

| 维度 | 计算逻辑 | 依赖游戏时长 | 影响程度 |
|------|---------|-------------|---------|
| KDA | (kills + assists) / deaths | ❌ 否 | 无影响 |
| 分均击杀 (KPM) | kills / 分钟数 | ✅ **是** | 高 |
| 分均经济 (GPM) | gold / 分钟数 | ✅ **是** | 高 |
| 参团率 | (kills + assists) / 团队总击杀 | ❌ 否 | 无影响 |
| 伤害转化比 | 伤害 / (团队总经济 * 1000) | ❌ 否 | 无影响 |
| 伤害占比 | 伤害 / 团队总伤害 | ❌ 否 | 无影响 |
| 承受伤害转化比 | 承受伤害 / (团队总经济 * 1000) | ❌ 否 | 无影响 |
| 承伤占比 | 承受伤害 / 团队总承受伤害 | ❌ 否 | 无影响 |
| **分均伤害** | 伤害 / 分钟数 | ✅ **是** | 高 |
| **分均承伤** | 承受伤害 / 分钟数 | ✅ **是** | 高 |
| 分均参团 | (kills + assists) / 分钟数 | ✅ **是** | 高 |
| 分均视野 | 视野得分 / 分钟数 | ✅ **是** | 高 |
| 分均插眼 | 插眼数 / 分钟数 | ✅ **是** | 高 |

### 1.3 问题场景模拟

**场景：导入数据时未填写游戏时长**

```typescript
// gameDuration = '' (空字符串)
const minutes = parseGameDuration(''); // 返回 0

// 计算分均击杀
const kpm = 25 / 0; // Infinity (异常值)

// 计算分均经济
const gpm = 65000 / 0; // Infinity (异常值)

// 计算分均伤害
const dpm = 28500 / 0; // Infinity (异常值)

// 雷达图显示异常（所有分均维度都是 Infinity）
```

**影响：**
- 雷达图维度计算结果为 `Infinity` 或 `NaN`
- 雷达图无法正常渲染
- 页面可能出现白屏或崩溃

#### 2.2.3 数据验证逻辑

**代码变更**：
```typescript
export function validateMatchInfo(data: MatchInfoData): ValidationResult {
  const errors: string[] = [];

  if (!data.redTeamName) {
    errors.push('红方战队名称不能为空');
  }
  if (!data.blueTeamName) {
    errors.push('蓝方战队名称不能为空');
  }
  if (data.gameNumber < 1 || data.gameNumber > 5) {
    errors.push('局数必须在1-5之间');
  }
  if (!data.gameStartTime) {
    errors.push('游戏开始时间不能为空');
  }
  // 恢复游戏时长验证
  if (!data.gameDuration) {
    errors.push('游戏时长不能为空');
  } else if (!isValidDurationFormat(data.gameDuration)) {
    errors.push('游戏时长格式错误，应为 MM:SS 格式');
  }
  if (!data.winner) {
    errors.push('获胜方不能为空');
  } else if (!['red', 'blue', 'Red', 'Blue', '红方', '蓝方'].includes(data.winner)) {
    errors.push('获胜方必须是 red 或 blue');
  }
  // BV号格式验证（如果填写）
  if (data.videoBvid && !isValidBVId(data.videoBvid)) {
    errors.push('视频BV号格式错误，应为 BVxxxxxxxxxx 格式');
  }
  return { valid: errors.length === 0, errors };
}
```

## 七、总结

### 7.1 核心结论

**游戏时长字段必须保留在导入模板中**，原因：
1. 雷达图 7 个维度依赖游戏时长计算
2. 没有游戏时长会导致分均维度计算异常
3. 前端虽然不展示游戏时长，但需要它进行计算

### 7.2 最终模板结构

| 列 | 字段 | 必填 | 说明 |
|---|---|---|---|
| A | 红方战队名 | 是 | |
| B | 蓝方战队名 | 是 | |
| C | 局数 | 是 | |
| D | 比赛时间 | 是 | |
| E | **游戏时长** | **是** | **用于雷达图计算** |
| F | 获胜方 | 是 | |
| G | MVP | 是 | |
| H | 视频BV号 | 否 | |

### 7.3 前端展示

- MatchInfoCard 不展示游戏时长
- 游戏时长仅用于雷达图维度计算（通过 RadarChart 组件）
