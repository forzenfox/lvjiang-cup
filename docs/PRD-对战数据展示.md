## 0. 术语表

| ------- | ----------------------------- |

| BO3     | Best of 3，三局两胜的赛制             |

## 1. 产品概览

### 1.1 项目背景

当前赛事系统中，对战信息展示较为基础，用户只能看到对阵双方和比赛结果，无法了解详细的对战数据。为了提升用户体验，增强赛事观赏性和专业性，需要增加详细的对战数据展示功能。当管理员上传了对战数据后，用户可以通过对战信息详情页面查看详细对战信息。

### 1.2 目标用户

- **赛事观众**：查看详细的对战数据，了解选手表现和比赛细节
- **参赛战队**：回顾比赛数据，分析战术和表现
- **赛事管理员**：上传和管理对战数据

### 1.3 产品价值

- 提升用户体验，满足观众对详细对战数据的需求
- 增强赛事专业性和观赏性
- 为战队和选手提供数据复盘支持
- 丰富赛事内容，增加用户停留时间

### 1.4 业务目标

### 2.2 详细功能描述

#### 2.2.1 对战数据按钮显示控制

**显示规则：**

- **显示条件**：只有当管理员为该对战上传了至少一条对战数据后，才显示"对战数据"按钮
- **隐藏条件**：未上传对战数据时，不显示该按钮（不显示占位符）
- **按钮位置**：对战信息详情页的操作区域，与其他操作按钮并列
- **按钮样式**：主色调按钮，图标 + 文字"对战数据"

**交互说明：**

- 鼠标悬停时显示手型指针和轻微放大效果
- 点击后打开新页面（新标签页）
- 按钮不可用时（如权限不足）显示禁用状态

#### 2.2.2 对战数据页面展示

**页面展示模式：**

对战数据详情页有两种展示模式，根据用户权限和操作方式自动切换：

| 模式 | 触发方式 | 权限要求 | 说明 |
|------|---------|---------|------|
| **只读模式** | 观众点击"对战数据"按钮进入 | 无 | 默认模式，仅展示数据，不可编辑 |
| **编辑模式** | 管理员在管理页面点击"编辑"按钮进入 | 管理员权限 | 可修改对战数据，修改后保存生效 |

**只读模式 - 页面结构：**

**模式切换说明：**

- 管理员在非编辑模式下，页面右上角显示"编辑"按钮
- 点击"编辑"按钮后，页面切换为编辑模式，所有可编辑字段解锁
- 编辑模式下，修改内容会实时标记（如字段边框变色提示已修改）
- 保存成功后，自动切换回只读模式
- 未保存退出时，提示"有未保存的修改，是否放弃？"

**加载状态：**

- 数据加载中显示骨架屏
- 加载失败显示错误提示和重试按钮
- 数据为空显示空状态提示

#### 2.2.3 多局对战数据支持

**赛制支持：**

- **BO1**：只显示 1 局数据，无对局切换器
- **BO3**：最多显示 3 局数据，支持 1-3 局切换
- **BO5**：最多显示 5 局数据，支持 1-5 局切换

**对局切换器设计：**

- **显示规则**：
  - 只有当对战数据包含多局时显示对局切换器
  - BO1 不显示切换器
  - BO3/BO5 根据实际上传的局数显示对应数量的切换按钮
- **切换按钮样式**：
  - 当前局：高亮显示（主色调背景）
  - 其他局：普通按钮样式
  - 已上传数据的局：显示完整样式
  - 未上传数据的局：显示禁用状态（灰色）
- **切换逻辑**：
  - 点击切换按钮立即加载对应局的数据
  - 切换时显示 loading 状态
  - 切换后 URL 更新（支持直接访问特定局）

**红蓝方说明：**

- 固定 **A 队（左侧队伍）为红色方**，**B 队（右侧队伍）为蓝色方**
- 红色方在英雄选择阶段有后手优势（最后一选），蓝色方有先手优势（第一选）
- 数据模型中需明确区分红蓝方，与英雄联盟比赛规则保持一致

**数据展示形式：**

参考 LPL 官方赛事数据展示风格，采用**对称式对战布局**：

- **左侧战队（红色方）**：选手头像 + 战队名 + 召唤师技能图标 + 英雄头像 + 战绩数据（击杀/死亡/助攻、金钱、补刀）
- **右侧战队（蓝色方）**：对称布局，与左侧战队数据对比展示
- **中间区域**：显示双方关键数据对比（补刀、金钱、击杀等）
- **展示字段**：选手信息、英雄选择、KDA、经济、补刀数
- **雷达图对比**：点击选手行展开六芒星雷达图，对比对位选手的 6 维度数据
- **移除字段**：不包含召唤师技能详情、天赋系统、详细出装列表（仅在选手行显示召唤师技能图标和英雄头像）

#### 2.2.5 六芒星雷达图对比

**功能说明：** 点击选手行展开六芒星雷达图，展示该选手与对位选手的 6 维度数据对比

**触发方式：**
- 点击选手数据行展开雷达图对比面板
- 点击其他选手行切换对位选手对比
- 点击同一选手行收起面板

**雷达图维度（按位置分类）：**

> **说明：** 雷达图的 6 个维度指标根据选手位置动态显示，不同位置展示不同维度的数据对比。

**上单（TOP）维度：**

**雷达图设计：**

- **形状**：正六边形，6 个顶点均匀分布（60°间隔）
- **同心层数**：5 层（从中心到外）
- **数据归一化**：每个维度的最大值取双方该维度的较大值，最小值为 0
- **配色方案**：
  - 左队/红色方：红色 `#f44336`（填充半透明，边框实线）
  - 右队/蓝色方：青色 `#00bcd4`（填充半透明，边框实线）
  - 网格线：`rgba(255, 255, 255, 0.1)`
  - 轴线：`rgba(255, 255, 255, 0.15)`
  - 维度标签：`#b0b0b0`

**交互说明：**
- 鼠标悬停雷达图维度区域显示具体数值 tooltip
- 显示双方该维度的对比数值
- 展开/收起动画时长 300ms

#### 2.3.4 雷达图数据计算规则

```typescript
// 雷达图数据计算函数
function computeRadarData(
  player: PlayerStat, 
  teamTotalKills: number, 
  teamTotalDamage: number
): RadarData {
  return {
    damageShare: player.damageShare 
      ?? (teamTotalDamage > 0 ? (player.damageDealt / teamTotalDamage) * 100 : 0),
    damageTakenPerDeath: player.deaths > 0 
      ? Math.round(player.damageTaken / player.deaths) 
      : 0,
    goldEarned: player.gold,
    creepScore: player.cs,
    killParticipation: player.killParticipation
      ?? (teamTotalKills > 0 ? ((player.kills + player.assists) / teamTotalKills) * 100 : 0),
    survival: player.kills - player.deaths  // 净击杀差，可为负值
  };
}
```

### 2.4 业务规则

#### 2.4.1 正常流程规则

- 对战数据按钮只在至少存在一条启用状态的对战数据时显示
- 对战数据页面默认展示第一局（game\_number=1）的数据
- 对局切换后，URL 自动更新为 `?game=X` 格式
- 选手数据按战队分组展示，同一战队内按位置排序（TOP→JUNGLE→MID→ADC→SUPPORT）

#### 2.4.2 边界规则

- **局数限制**：
  - BO1：只允许上传 1 局数据
  - BO3：最多上传 3 局数据
  - BO5：最多上传 5 局数据
  - 超出限制时提示"局数超出赛制限制"
- **数据完整性**：
  - 单局数据必须包含基础信息（局数、获胜方、游戏时长）
  - 选手数据至少包含 5 名选手（每队 5 人）
  - 数据不完整时显示"数据加载中"或"部分数据暂不可用"
- **权限控制**：
  - 所有用户可查看对战数据
  - 只有管理员可上传/编辑/删除对战数据
- **状态控制**：
  - 禁用状态（status=0）的数据不对外展示
  - 只有管理员可看到禁用状态的数据

#### 2.4.3 异常处理

- **数据加载失败**：
  - 自动重试 3 次（间隔 1 秒）
  - 重试失败后显示"数据加载失败，请刷新重试" + 刷新按钮
- **对战数据不存在**：
  - 显示空状态："该对战暂无数据"
  - 提供返回按钮
- **局数不存在**：
  - 访问不存在的局数（如 BO3 的第 4 局）时，显示"该对局数据不存在"
  - 自动跳转到第一局
- **网络错误**：
  - 显示"网络连接异常，请检查网络设置"
  - 提供重试按钮

```typescript
      loader: ({ params }) => loadMatchGameData(matchId, params.gameNumber)
    }
  ]
}
```

### 9.5 FAQ

| 问题                  | 回答                                  |
| ------------------- | ----------------------------------- |
| 为什么 BO3 只显示了 2 局数据？ | 可能第 3 局尚未进行或管理员尚未上传数据，未上传数据的局显示禁用状态 |
| 对战数据按钮为什么不显示？       | 只有管理员上传了至少一条对战数据后才会显示该按钮            |
| 如何查看其他局的数据？         | 使用页面上方的对局切换器，点击对应局数按钮即可切换           |
| 移动端选手数据看不清怎么办？    | 移动端采用卡片式展示，支持左右滑动切换选手              |
| 雷达图数据加载慢怎么办？        | 雷达图数据量较大时会进行采样处理，如仍然加载缓慢，建议检查网络连接    |
| 如何上传对战数据？            | 管理员在管理后台左侧菜单进入"对战数据"模块，在已结束的对战记录上点击"对战数据导入"按钮 |
| 为什么有些对战记录没有编辑按钮？  | 只有已导入对战数据的记录才显示编辑按钮，未导入数据的记录只显示导入按钮    |
| 编辑模式如何进入？            | 管理员在对战数据管理页面点击已结束且已导入数据的记录上的"编辑"按钮，自动进入编辑模式 |
| 对战数据详情页有几种展示模式？   | 两种模式：只读模式（观众默认进入）和编辑模式（管理员需鉴权）     |
| 上传时是否需要填写赛事 ID 和对战 ID？ | 不需要，上传时已关联当前对战，数据文件只需填写战队名称、选手昵称和英雄名 |
| 战队名称可以重复吗？            | 不可以，战队名称必须唯一（数据库有 UNIQUE 约束），确保导入时能准确匹配 |
| 选手昵称和英雄名如何匹配？       | 支持模糊匹配，忽略大小写和前后空格；英雄名需匹配系统英雄字典中的中文名称 |
