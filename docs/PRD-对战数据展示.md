# 对战数据展示功能 PRD

## 0. 术语表

| 术语 | 定义 |
|------|------|
| BO1 | Best of 1，一局定胜负的赛制 |
| BO3 | Best of 3，三局两胜的赛制 |
| BO5 | Best of 5，五局三胜的赛制 |
| 对局 | 单场游戏，一局对战 |
| 对战系列 | 一个完整的 BO 系列赛，可能包含 1-N 局对局 |
| 对战数据 | 管理员上传的单场对战详细信息，包括选手表现、英雄选择等 |
| 对战信息详情页 | 展示对战基本信息的详情页面，包括对阵双方、比赛时间、赛制等 |

## 1. 产品概览

### 1.1 项目背景

当前赛事系统中，对战信息展示较为基础，用户只能看到对阵双方和比赛结果，无法了解详细的对战数据。为了提升用户体验，增强赛事观赏性和专业性，需要增加详细的对战数据展示功能。当管理员上传了对战数据后，用户可以通过对战信息详情页面查看详细对战信息。

### 1.2 目标用户

- **赛事观众**：查看详细的对战数据，了解选手表现和比赛细节
- **参赛战队**：回顾比赛数据，分析战术和表现
- **赛事管理员**：上传和管理对战数据

### 1.3 产品价值

- 提升用户体验，满足观众对详细对战数据的需求
- 增强赛事专业性和观赏性
- 为战队和选手提供数据复盘支持
- 丰富赛事内容，增加用户停留时间

### 1.4 业务目标

| 指标 | 当前值 | 目标值 | 测量周期 | 计算方式 | 目标依据 |
|------|--------|--------|----------|----------|----------|
| 对战详情页平均停留时长 | 30 秒 | 60 秒 (+100%) | 上线后 30 天 | 用户在对战详情页停留总时长/访问次数 | 行业参考：数据展示类页面通常可提升停留时长 80%-120% |
| 对战数据查看率 | 0% | 40% | 上线后 14 天 | 点击"对战数据"按钮的用户数/对战详情页访问用户数 | 推算：预估对战详情页日活 1000 × 数据查看意愿 40% ≈ 400 次/天 |
| 对战详情页 PV | 基准值 | +20% | 上线后 30 天 | 页面浏览量对比上线前 | 数据内容带来的回访增量 |

### 1.5 用户故事

| 编号 | 用户故事 | 验收条件 |
|------|----------|----------|
| US-001 | 作为观众，我希望在对战信息详情页看到"对战数据"按钮，以便快速访问详细对战数据 | 对战数据按钮在满足条件时显示；点击后打开新页面 |
| US-002 | 作为观众，我希望查看 BO1/BO3/BO5 等不同赛制的对战数据，以便了解完整的系列赛情况 | 支持多局对战数据展示；正确显示局数信息 |
| US-003 | 作为观众，我希望在不同对局之间切换查看，以便对比各局数据 | 支持对局切换；切换流畅 |
| US-004 | 作为管理员，我希望上传对战数据后控制其显示，以便在数据完整前不对外展示 | 只有上传数据后才显示按钮；支持批量上传 |

## 2. 核心功能

### 2.1 功能概览

| 功能模块 | 功能描述 | 优先级 |
|----------|----------|--------|
| 对战数据按钮显示控制 | 只有在管理员上传了对战数据后，对战信息详情页才显示"对战数据"按钮 | P0 |

### 2.2 详细功能描述

#### 2.2.1 对战数据按钮显示控制

**显示规则：**
- **显示条件**：只有当管理员为该对战上传了至少一条对战数据后，才显示"对战数据"按钮
- **隐藏条件**：未上传对战数据时，不显示该按钮（不显示占位符）
- **按钮位置**：对战信息详情页的操作区域，与其他操作按钮并列
- **按钮样式**：主色调按钮，图标 + 文字"对战数据"

**交互说明：**
- 鼠标悬停时显示手型指针和轻微放大效果
- 点击后打开新页面（新标签页或当前页跳转，可配置）
- 按钮不可用时（如权限不足）显示禁用状态

#### 2.2.2 对战数据页面展示

**页面结构：**

**加载状态：**
- 数据加载中显示骨架屏
- 加载失败显示错误提示和重试按钮
- 数据为空显示空状态提示

#### 2.2.3 多局对战数据支持

**赛制支持：**
- **BO1**：只显示 1 局数据，无对局切换器
- **BO3**：最多显示 3 局数据，支持 1-3 局切换
- **BO5**：最多显示 5 局数据，支持 1-5 局切换

**对局切换器设计：**
- **显示规则**：
  - 只有当对战数据包含多局时显示对局切换器
  - BO1 不显示切换器
  - BO3/BO5 根据实际上传的局数显示对应数量的切换按钮
- **切换按钮样式**：
  - 当前局：高亮显示（主色调背景）
  - 其他局：普通按钮样式
  - 已上传数据的局：显示完整样式
  - 未上传数据的局：显示禁用状态（灰色）
- **切换逻辑**：
  - 点击切换按钮立即加载对应局的数据
  - 切换时显示 loading 状态
  - 切换后 URL 更新（支持直接访问特定局）

```typescript
  gold: number;                 // 总经济
  towers: number;               // 推塔数
  dragons: number;              // 控龙数
  barons: number;               // 控 Baron 数

}

// 选手对战数据
interface PlayerStat {
  id: number;
  playerId: number;
  playerName: string;
  teamId: number;
  teamName: string;
  position: string;             // 位置
  championId: number;
  championName: string;
  kills: number;
  deaths: number;
  assists: number;
  cs: number;
  gold: number;
  damageDealt: number;
  damageTaken: number;
  items: number[];              // 装备 ID 列表
  level: number;
  visionScore: number;
  firstBlood: boolean;
  mvp: boolean;
  kda: string;                  // KDA 字符串 "8/2/12"

// 对战系列信息（用于对局切换）
interface MatchSeriesInfo {
```

### 2.4 业务规则

#### 2.4.1 正常流程规则

- 对战数据按钮只在至少存在一条启用状态的对战数据时显示
- 对战数据页面默认展示第一局（game_number=1）的数据
- 对局切换后，URL 自动更新为 `?game=X` 格式
- 选手数据按战队分组展示，同一战队内按位置排序（TOP→JUNGLE→MID→ADC→SUPPORT）

#### 2.4.2 边界规则

- **局数限制**：
  - BO1：只允许上传 1 局数据
  - BO3：最多上传 3 局数据
  - BO5：最多上传 5 局数据
  - 超出限制时提示"局数超出赛制限制"
- **数据完整性**：
  - 单局数据必须包含基础信息（局数、获胜方、游戏时长）
  - 选手数据至少包含 5 名选手（每队 5 人）
  - 数据不完整时显示"数据加载中"或"部分数据暂不可用"
- **权限控制**：
  - 所有用户可查看对战数据
  - 只有管理员可上传/编辑/删除对战数据
- **状态控制**：
  - 禁用状态（status=0）的数据不对外展示
  - 只有管理员可看到禁用状态的数据

#### 2.4.3 异常处理

- **数据加载失败**：
  - 自动重试 3 次（间隔 1 秒）
  - 重试失败后显示"数据加载失败，请刷新重试" + 刷新按钮
- **对战数据不存在**：
  - 显示空状态："该对战暂无数据"
  - 提供返回按钮
- **局数不存在**：
  - 访问不存在的局数（如 BO3 的第 4 局）时，显示"该对局数据不存在"
  - 自动跳转到第一局
- **网络错误**：
  - 显示"网络连接异常，请检查网络设置"
  - 提供重试按钮

### 3.3 交互设计

#### 3.3.1 对局切换交互

- **切换方式**：
  - 点击对局按钮立即切换
  - 切换时显示 loading 状态（300ms）
  - 切换动画：淡入淡出（200ms）
- **URL 更新**：
  - 切换后 URL 更新为 `/match/{id}/games?game=X`
  - 支持直接访问特定局（带 query 参数）

- **数据采样**：经济曲线等时间序列数据按固定间隔采样（如每分钟一个点）
- **防抖节流**：缩放/拖拽事件防抖 100ms

#### 4.3.3 表格优化

- **虚拟滚动**：大数据量时使用虚拟滚动
- **分页加载**：支持分页展示（每页 10 条）

### 9.4 错误码定义

```
0 - 成功
40001 - 参数校验失败
40002 - 局数超出赛制限制
40003 - 对战数据不存在
40004 - 权限不足
40100 - 未登录或 token 过期
40300 - 无操作权限
40400 - 资源不存在
50000 - 服务器内部错误
```

### 9.5 FAQ

| 问题 | 回答 |
|------|------|
| 为什么 BO3 只显示了 2 局数据？ | 可能第 3 局尚未进行或管理员尚未上传数据，未上传数据的局显示禁用状态 |
| 对战数据按钮为什么不显示？ | 只有管理员上传了至少一条对战数据后才会显示该按钮 |
| 如何查看其他局的数据？ | 使用页面上方的对局切换器，点击对应局数按钮即可切换 |
| 移动端表格看不清怎么办？ | 移动端表格支持横向滚动，可左右滑动查看完整数据；或切换到卡片模式查看 |
| 图表数据加载慢怎么办？ | 图表数据量较大时会进行采样处理，如仍然加载缓慢，建议检查网络连接 |
