# 批量导入战队与队员信息 PRD

## 1. 文档概述

### 1.1 产品背景

当前驴酱杯赛事网站的管理后台中，战队和队员信息只能逐个手动创建和编辑。每支战队需要填写战队名称、队标、参赛宣言，以及5名队员（上单、打野、中单、ADC、辅助）各自的昵称、游戏ID、头像、评分、常用英雄等信息。当参赛战队数量较多（最多16支）时，管理员需要反复操作，流程繁琐、效率低下，且容易出错。

为简化管理员操作、提升效率，需要支持通过 Excel 文件批量导入战队信息和队员信息，使管理员能够一次性完成多支战队及其队员数据的录入。

### 1.2 产品目标

- 降低管理员录入战队和队员信息的操作成本，提升管理效率
- 支持 Excel 模板下载 → 填写 → 上传导入的标准化流程
- 提供完善的导入校验和错误反馈机制，确保数据质量
- 兼容现有手动管理模式，批量导入与手动编辑互不冲突

### 1.3 目标用户

- **赛事管理员**：需要快速录入大量战队和队员信息

### 1.4 适用范围

- 前端：管理后台战队管理页面（新增批量导入入口）
- 后端：新增批量导入 API 接口、Excel 解析与校验逻辑

#### 2.2.4 文件校验规则

**前端校验：**

| 校验项 | 规则 | 错误提示 |
|--------|------|----------|
| 文件格式 | 仅支持 `.xlsx` | 仅支持 .xlsx 格式的 Excel 文件 |
| 文件大小 | 不超过 5MB | 文件大小不能超过 5MB |
| 文件是否为空 | 文件内容不为空 | 上传的文件内容为空 |

**后端校验：**

| 校验项 | 规则 | 错误提示 |
|--------|------|----------|
| Sheet名称 | 包含"战队与队员信息导入" Sheet | 未找到正确的数据Sheet |
| 列头匹配 | 列头与模板一致 | 列头格式不匹配，请使用标准模板 |
| 数据行数 | 至少1行数据，最多16行 | 导入数据行数超出范围（1-16行） |
| 战队名称 | 必填，不可重复（文件内+数据库内） | 第X行：战队名称不能为空 / 战队名称"XXX"重复 |
| 队标URL | 合法URL格式（如填写） | 第X行：队标URL格式不正确 |
| 头像URL | 合法URL格式（如填写） | 第X行：上单头像URL格式不正确 |
| 评分 | 0-100整数（如填写） | 第X行：上单评分必须为0-100的整数 |
| 是否队长 | "是"或"否"（如填写） | 第X行：上单是否队长只能填写"是"或"否" |
| 队长数量 | 每支战队最多1名队长 | 第X行：战队"XXX"设置了多名队长，每队仅可设置1名 |
| 战队总数 | 导入后战队总数不超过16 | 导入后战队总数将超过16支上限 |

### 2.3 导入模式

#### 2.3.1 仅新增模式

- 跳过数据库中已存在的同名战队
- 仅导入数据库中不存在的战队
- 导入结果中标注哪些战队被跳过

#### 2.3.2 新增并更新模式

- 对于数据库中已存在的同名战队：更新战队信息和队员信息（全量覆盖队员数据）
- 对于数据库中不存在的战队：新增战队
- 更新时保留原战队的 ID，不破坏关联的比赛数据

### 2.5 导入后手动补充操作

#### 2.5.1 功能描述

由于批量导入仅支持设置外链形式的队标和队友头像，管理员在导入完成后，可能需要手动上传本地图片文件。

#### 2.5.2 操作指引

导入结果反馈中，明确提示哪些战队/队员使用了外链URL，管理员可：
1. 在战队管理页面找到对应战队
2. 展开战队详情，进入编辑模式
3. 通过上传组件替换队标或队员头像为本地文件
4. 保存修改

## 3. 数据结构设计

### 3.1 Excel 导入数据映射

Excel 中的数据将映射到现有的数据库表结构，不新增数据表。

**战队表（teams）映射：**

| Excel 列 | 数据库字段 | 映射规则 |
|----------|-----------|---------|
| team_name | name | 直接映射 |
| team_logo_url | logo | 直接映射（外链URL） |
| team_battle_cry | battle_cry | 直接映射 |

**队员表（team_members）映射：**

每个位置（TOP/JUNGLE/MID/ADC/SUPPORT）对应5列数据，映射规则相同：

| Excel 列前缀 | 数据库字段 | 映射规则 |
|-------------|-----------|---------|
| {position}_nickname | nickname | 直接映射 |
| {position}_game_id | game_id | 直接映射 |
| {position}_avatar_url | avatar_url | 直接映射（外链URL） |
| {position}_rating | rating | 直接映射，默认60 |
| {position}_is_captain | is_captain | "是"→1，"否"→0，默认0 |

### 3.2 不支持导入的字段

以下字段不支持通过 Excel 导入，需在导入后手动操作：

| 字段 | 原因 | 手动操作方式 |
|------|------|-------------|
| 队标本地上传 | Excel 不支持嵌入图片文件 | 战队管理 → 编辑战队 → 上传队标 |
| 队员头像本地上传 | Excel 不支持嵌入图片文件 | 战队管理 → 展开战队 → 编辑队员 → 上传头像 |
| 常用英雄（champion_pool） | 英雄选择器交互复杂，Excel 难以准确表达 | 战队管理 → 展开战队 → 编辑队员 → 选择英雄 |
| 实力等级（level） | 与评分有逻辑关联，避免冲突 | 战队管理 → 展开战队 → 编辑队员 → 设置等级 |
| 直播间链接（live_url） | 需要格式化处理（斗鱼链接前缀拼接） | 战队管理 → 展开战队 → 编辑队员 → 填写直播间 |
| 个人简介（bio） | 长文本内容，Excel 填写体验差 | 战队管理 → 展开战队 → 编辑队员 → 填写简介 |

## 5. 交互设计说明

### 5.1 战队管理页面变更

#### 5.1.1 操作栏变更

在现有"刷新"和"添加战队"按钮旁，新增两个按钮：

```
┌──────────────────────────────────────────────────────────────┐
│  战队管理                                                     │
│                                                               │
│  [🔄 刷新]  [📥 下载模板]  [📤 批量导入]  [➕ 添加战队]      │
└──────────────────────────────────────────────────────────────┘
```

**按钮样式：**
- "下载模板"：outline 变体，与"刷新"按钮风格一致
- "批量导入"：主色变体，与"添加战队"按钮风格一致，使用渐变背景

#### 5.1.2 导入流程

```
graph TD
    A[点击批量导入] --> B[弹出导入对话框]
    B --> C[选择文件/拖拽上传]
    C --> D{前端校验}
    D -->|格式/大小错误| E[提示错误]
    D -->|校验通过| F[上传至后端]
    F --> G{后端校验}
    G -->|校验失败| H[显示错误列表]
    H --> I[下载错误报告 / 重新上传]
    G -->|校验通过| J[执行导入]
    J --> K[显示导入结果]
    K --> L{是否有外链项?}
    L -->|是| M[提示手动上传]
    L -->|否| N[完成]
    M --> N
```

### 5.2 导入进度反馈

- 上传文件时显示上传进度条
- 后端解析和导入时显示 loading 状态（"正在解析数据..." / "正在导入数据..."）
- 导入过程中禁止页面其他操作

### 5.3 导入后页面刷新

- 导入成功后自动刷新战队列表
- 新导入的战队在列表中高亮显示（3秒后恢复正常）

## 6. 边界条件与异常处理

### 6.1 数据冲突处理

| 场景 | 处理方式 |
|------|---------|
| 战队名称在文件内重复 | 校验阶段报错，不执行导入 |
| 战队名称与数据库重复（仅新增模式） | 跳过该战队，在结果中标注 |
| 战队名称与数据库重复（新增并更新模式） | 更新该战队信息，覆盖队员数据 |
| 同一战队设置多名队长 | 校验阶段报错，不执行导入 |
| 导入后战队总数超过16 | 校验阶段报错，不执行导入 |

### 6.2 文件异常处理

| 场景 | 处理方式 |
|------|---------|
| 文件格式不是 .xlsx | 前端校验拦截，提示"仅支持 .xlsx 格式" |
| 文件损坏无法解析 | 后端返回错误，提示"文件解析失败，请检查文件是否损坏" |
| 文件内容为空（只有列头） | 后端返回错误，提示"文件中没有有效数据" |
| 文件大小超过5MB | 前端校验拦截，提示"文件大小不能超过 5MB" |
| 上传过程中网络中断 | 提示"网络异常，请重试"，不执行任何数据变更 |

### 6.3 数据格式异常

| 场景 | 处理方式 |
|------|---------|
| 评分不是整数 | 校验报错："评分必须为0-100的整数" |
| 评分超出范围 | 校验报错："评分必须为0-100的整数" |
| URL格式不正确 | 校验报错："URL格式不正确" |
| 是否队长填写非"是/否" | 校验报错："是否队长只能填写'是'或'否'" |
| 战队名称超长 | 校验报错："战队名称不能超过20个字符" |
| 必填字段为空 | 校验报错："战队名称不能为空" |

### 6.4 并发处理

- 导入操作期间锁定战队数据，禁止同时进行其他战队管理操作
- 如果已有导入任务正在执行，提示"当前有导入任务正在进行，请稍后重试"

## 7. 技术方案要点

### 7.1 后端技术选型

| 技术 | 用途 | 说明 |
|------|------|------|
| xlsx / exceljs | Excel 文件解析与生成 | 用于解析上传的 Excel 文件和生成模板/错误报告 |
| class-validator | 数据校验 | 复用现有 DTO 校验逻辑 |
| multer | 文件上传 | 复用现有上传中间件 |

### 7.2 导入流程（后端）

1. 接收上传的 Excel 文件
2. 解析 Excel 文件，提取数据行
3. 逐行校验数据（格式校验 + 业务校验）
4. 若存在校验错误，收集所有错误并返回，不执行任何数据写入
5. 校验全部通过后，开启数据库事务
6. 逐行执行导入（新增或更新）
7. 提交事务
8. 清除相关缓存
9. 返回导入结果

### 7.3 事务安全

- 导入操作使用数据库事务，确保原子性
- 任一行导入失败则整体回滚（校验阶段已排除错误，此处为运行时异常保障）
- 更新模式下先删除旧队员再插入新队员，确保数据一致性

### 7.4 性能考虑

- Excel 文件限制最大 16 行数据（对应16支战队），数据量小，无需分批处理
- 导入完成后清除战队相关缓存（`teams:all` 和 `team:{id}`）
