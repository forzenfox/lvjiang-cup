# 主页面赛事视频模块 PRD

## 1. 产品概览

### 1.1 项目背景
随着赛事内容的不断丰富，用户对于赛事相关视频的需求日益增长。为了提升用户体验，增强网站的内容吸引力，需要在主页面新增一个视频模块，展示赛事相关的视频内容。

### 1.2 目标用户
- **普通观众**：对赛事感兴趣，希望快速了解赛事相关视频内容
- **赛事粉丝**：关注特定赛事，希望方便地观看赛事回放、精彩集锦等内容

### 1.3 产品价值
- 提升主页面的内容丰富度和视觉吸引力
- 为用户提供便捷的赛事视频观看入口
- 增加网站的用户停留时间和页面浏览量
- 满足运营团队对赛事内容推广的需求

### 1.4 业务目标
| 指标 | 当前值 | 目标值 | 测量周期 | 计算方式 |
|------|--------|--------|---------|---------|
| 主页用户平均停留时长 | 2 分钟 | 3 分钟 (+50%) | 上线后 30 天 | 用户在主页停留总时长/访问次数 |
| 视频播放量 | 0 | 500 次/天 | 上线后 7 天 | 每日视频播放次数统计 |
| 视频点击率 | - | 15% | 上线后 14 天 | 视频播放次数/视频模块曝光次数 |
| 主页 PV | 基准值 | +10% | 上线后 30 天 | 页面浏览量对比上线前 |

### 1.5 竞品分析
| 竞品 | 特点 | 可借鉴点 |
|------|------|---------|
| LPL 官网 | 视频轮播 + 分类标签 + 热度排序 | 分类筛选、热门推荐 |
| LCK 官网 | 视频按赛事阶段分类 | 赛事分组展示 |
| 王者荣耀赛事 | 精彩集锦自动轮播 | 自动轮播功能 |
| 参考点 | - | 分类筛选、热门推荐、播放历史、分享功能 |

## 2. 核心功能

### 2.1 功能概览
| 功能模块 | 功能描述 |
|---------|--------|
| 视频轮播展示 | 三屏布局（PC 端）：中间主视频自动播放，左右展示相邻视频缩略图，支持点击箭头或缩略图切换，最多展示 10 个视频 |
| B 站视频嵌入 | 通过 B 站嵌入式播放器播放视频内容 |
| 后台管理配置 | 管理员通过后台页面配置视频链接 |
| 响应式设计 | 支持 PC 端（三屏布局）和移动端（单屏布局）适配 |

### 2.2 详细功能描述

#### 2.2.1 视频轮播展示
- **三屏布局**：中间展示主视频，左右展示相邻视频的缩略图
- **自动播放**：中间主视频自动播放，切换视频后自动播放新选择的视频
- **左右切换**：用户可通过点击左右箭头或左右缩略图切换视频
- **视频预览**：主视频显示完整播放器，左右缩略图显示视频缩略图和标题
- **数量限制**：最多展示 10 个视频，超过 10 个时可循环滚动

#### 2.2.2 B 站视频嵌入
- **嵌入式播放器**：使用 B 站提供的嵌入式播放器代码
- **播放控制**：支持播放/暂停、音量调节、全屏等基本控制
- **清晰度选择**：根据 B 站播放器支持的清晰度自动适配
- **弹幕功能**：默认开启弹幕功能，与 B 站保持一致

#### 2.2.3 后台管理配置
- **视频链接管理**：管理员可添加、编辑、删除视频链接
- **排序功能**：可调整视频的展示顺序
- **状态管理**：可启用/禁用视频展示
- **链接验证**：验证 B 站链接的有效性

#### 2.2.4 响应式设计
- **PC 端**：三屏布局，中间主视频，左右缩略图，支持点击箭头或缩略图切换
- **移动端**：单屏布局，只展示主视频，支持左右滑动切换
- **适配范围**：移动端适配仅针对网站主页面，后台管理页面仅适配 PC 端
- **适配断点**：针对不同屏幕尺寸进行适配

### 2.3 数据结构定义

#### 2.3.1 视频数据表结构
| 字段名 | 类型 | 必填 | 默认值 | 说明 | 示例 |
|--------|------|------|--------|------|------|
| id | string | 是 | - | 视频唯一标识 | "video_001" |
| title | string | 是 | - | 视频标题（最多 50 字符） | "2025 绿江杯决赛 G5 集锦" |
| bvid | string | 是 | - | B 站视频 ID | "BV1xx411c7mD" |
| cover_url | string | 否 | null | 封面图 URL（可选，默认使用 B 站封面） | "https://example.com/cover.jpg" |
| order | number | 是 | 0 | 排序值，越小越靠前 | 1 |
| status | enum | 是 | 1 | 状态：0-禁用，1-启用 | 1 |
| created_at | datetime | 是 | - | 创建时间 | 2025-01-15 10:00:00 |
| updated_at | datetime | 是 | - | 更新时间 | 2025-01-15 10:00:00 |
| created_by | string | 是 | - | 创建人（管理员 ID） | "admin_001" |

#### 2.3.2 前端展示数据结构
```typescript
interface VideoItem {
  id: string;           // 视频 ID
  title: string;        // 视频标题
  bvid: string;         // B 站视频 ID
  coverUrl?: string;    // 封面图 URL（可选）
  order: number;        // 排序值
  status: 0 | 1;        // 状态
}

interface VideoListResponse {
  code: number;         // 响应码：0-成功，其他 - 失败
  data: {
    list: VideoItem[];  // 视频列表
    total: number;      // 总数
  };
  message?: string;     // 错误消息（失败时）
}
```

### 2.4 业务规则

#### 2.4.1 正常流程规则
- 视频添加后默认为启用状态（status=1）
- 排序值相同时，按创建时间倒序排列
- 前端最多展示 10 个启用状态的视频
- 后台可存储超过 10 个视频，但超出部分不展示
- 视频切换后，新视频自动播放

#### 2.4.2 边界规则
- **链接格式校验**：
  - 必须为 B 站链接（支持 www.bilibili.com、b23.tv 短链接）
  - 支持格式：`https://www.bilibili.com/video/BV{bvid}` 或 `https://b23.tv/{shortCode}`
  - 自动从完整链接中提取 bvid
- **标题限制**：最多 50 个字符（包含中文、英文、数字）
- **重复检测**：同一 bvid 不可重复添加
- **数量限制**：最多启用 10 个视频，超出时提示"最多可启用 10 个视频"
- **排序范围**：order 值范围 0-9999

#### 2.4.3 异常处理
- **B 站视频失效**：
  - 前端显示"视频已失效"占位图
  - 后台标记为异常状态，发送通知给管理员
- **网络加载失败**：
  - 自动重试 3 次（间隔 1 秒）
  - 重试失败后显示"网络连接异常，请刷新重试"
- **超出数量限制**：
  - 后台提示"最多可启用 10 个视频，请先禁用部分视频"
  - 不允许继续启用新视频
- **无效链接**：
  - 提交时即时校验，显示"链接格式不正确，请输入有效的 B 站视频链接"
  - 不允许保存

## 3. 页面设计

### 3.1 设计风格
- **整体风格**：参考 LPL 官方页面设计，保持与现有网站风格一致
- **色彩方案**：使用网站主色调，确保视觉一致性
- **字体**：使用网站统一字体，保证品牌识别度

### 3.2 页面布局
- **位置**：主页面核心位置，用户进入网站后一眼可见
- **尺寸**：宽度与主页面一致，高度适中，确保不影响其他内容展示
- **布局**：
  - 顶部：模块标题（如"赛事精彩视频"）
  - 中部：视频轮播区域，采用三屏布局
    - 左侧：上一个视频的缩略图预览
    - 中间：当前主视频播放器，自动播放
    - 右侧：下一个视频的缩略图预览
    - 左右两侧：滚动箭头按钮
  - 底部：查看更多链接（可选）

### 3.3 交互设计
- **切换交互**：
  - PC 端：点击左右箭头按钮或左右缩略图切换视频
  - 移动端：左右滑动屏幕切换视频
- **播放交互**：中间主视频自动播放，切换后自动播放新选择的视频
- **悬停效果**：PC 端鼠标悬停在左右缩略图上时有轻微放大效果
- **点击缩略图**：点击左右缩略图可直接切换到对应视频

### 3.4 视觉元素
- **主视频播放器**：中间位置，尺寸最大，显示完整的 B 站播放器
- **左右缩略图**：
  - 位置：主视频的左右两侧
  - 尺寸：比主视频小，半透明效果
  - 内容：显示视频缩略图和标题
- **滚动指示器**：显示当前播放视频位置和总视频数量
- **控制按钮**：左右滚动箭头，样式与整体设计风格一致

### 3.5 状态设计

#### 3.5.1 加载状态
- **首屏加载**：显示骨架屏（灰色占位块）
- **视频切换**：显示 Loading 旋转动画（时长 300ms）
- **图片加载**：渐进式加载，先显示模糊缩略图

#### 3.5.2 空状态
- **无视频时**：显示空状态插图 + 文案"暂无视频内容"
- **后台管理**：显示"添加第一个视频"引导按钮

#### 3.5.3 错误状态
- **播放器加载失败**：显示"视频加载失败，请刷新重试" + 刷新按钮
- **网络错误**：显示"网络连接异常，请检查网络设置"
- **B 站视频失效**：显示"视频已下架或不存在" + 联系管理员入口

### 3.6 交互细节

| 交互动作 | 动画时长 | 缓动函数 | 说明 |
|---------|---------|---------|------|
| 视频切换滑动 | 400ms | ease-out | 左右滑动过渡动画 |
| 缩略图悬停放大 | 200ms | ease-in-out | 放大比例 1.05 倍 |
| 点击缩略图切换 | 300ms | ease | 切换动画 |
| 自动轮播间隔 | 5000ms | linear | 5 秒自动切换（可选配置） |
| 错误提示显示 | 200ms | ease-in | Toast 提示动画 |

## 4. 技术实现

### 4.1 前端实现
- **技术栈**：使用现有网站的前端技术栈
- **轮播组件**：实现左右滚动的轮播组件，支持触摸滑动
- **B 站播放器集成**：使用 B 站提供的嵌入式播放器代码
  - 嵌入方式：使用 iframe 标签嵌入 B 站播放器
  - 参数配置：根据需要配置播放器参数，如自动播放、弹幕等

### 4.2 后端实现
- **视频管理 API**：提供视频链接的增删改查接口
- **数据存储**：存储视频链接、标题、排序等信息
- **链接验证**：验证 B 站链接的有效性，确保播放器能正常加载

### 4.3 后台管理
- **管理界面**：在现有后台系统中添加视频管理模块
- **操作权限**：只有管理员角色可操作视频管理功能
- **操作流程**：
  1. 管理员登录后台
  2. 进入视频管理模块
  3. 添加/编辑/删除视频链接
  4. 调整视频排序
  5. 保存配置

### 4.4 接口需求

#### 4.4.1 获取视频列表（前端）
- **URL**: `GET /api/streams/videos`
- **参数**: 
  ```typescript
  {
    limit?: number,     // 每页数量，默认 10
    offset?: number     // 偏移量，默认 0
  }
  ```
- **响应**:
  ```json
  {
    "code": 0,

      "list": [
        {
          "id": "video_001",

          "bvid": "BV1xx411c7mD",
          "cover_url": "https://example.com/cover.jpg",
          "order": 1,
          "status": 1

      ],
      "total": 10

    "message": "success"
  }
  ```

#### 4.4.2 添加视频（后台）
- **URL**: `POST /api/admin/videos`
- **请求体**:

  ```json
  {

    "data": {
      "id": "video_001"

  }
  ```

#### 4.4.3 更新视频（后台）
- **URL**: `PUT /api/admin/videos/:id`


- **URL**: `DELETE /api/admin/videos/:id`
- **响应**:

### 4.5 性能优化方案

#### 4.5.1 懒加载策略
- **首屏加载**：仅加载当前视频及左右相邻视频（共 3 个）
- **视口加载**：其他视频进入视口前 500px 时开始加载
- **iframe 懒加载**：使用 `loading="lazy"` 属性

#### 4.5.2 预加载策略
- **悬停预加载**：鼠标悬停缩略图超过 200ms 时，预加载下一个视频 iframe
- **空闲预加载**：使用 `requestIdleCallback` 在浏览器空闲时预加载

#### 4.5.3 防抖节流
- **切换防抖**：视频切换操作防抖 300ms，避免频繁切换
- **滚动节流**：滚动事件节流 100ms

#### 4.5.4 缓存策略
- **接口缓存**：视频列表接口数据缓存 5 分钟（localStorage）
- **播放器缓存**：使用 B 站播放器的内置缓存
- **图片缓存**：封面图设置长期缓存（Cache-Control: max-age=31536000）

#### 4.5.5 渲染优化
- **限制同时渲染**：最多同时渲染 3 个 iframe（当前 + 左右）
- **离屏销毁**：超出可视范围的 iframe 从 DOM 移除
- **使用 CSS transform**：动画使用 transform 而非 left/top 属性

## 5. 移动端适配

### 5.1 适配策略
- **响应式布局**：使用 CSS 媒体查询实现不同屏幕尺寸的适配
- **触摸优化**：针对移动设备的触摸操作进行优化
- **性能优化**：确保在移动设备上加载流畅
- **适配范围**：移动端适配仅针对网站主页面，后台管理页面仅适配 PC 端

### 5.2 适配断点
- **PC 端**：屏幕宽度 ≥ 1024px
- **平板端**：768px ≤ 屏幕宽度 < 1024px
- **移动端**：屏幕宽度 < 768px

### 5.3 移动端特殊处理
- **布局调整**：移动端采用单屏布局，只展示主视频
- **视频卡片**：每次只展示一个视频卡片
- **滚动方式**：PC 端三屏布局（点击左右箭头或缩略图切换），移动端单屏布局（支持左右滑动切换）
- **播放器尺寸**：根据屏幕宽度自动调整播放器大小
- **加载优化**：优先加载当前视频，其他视频延迟加载
- **自动播放限制**：移动端 Safari 禁用自动播放，需用户手动点击

### 7.1 风险矩阵

|------|------|------|--------|---------|--------|

### 7.2 回滚方案

#### 7.2.1 快速回滚
- **功能开关**：通过配置中心控制视频模块开关，出现问题时一键关闭
- **灰度发布**：先对 10% 用户开放，验证无误后全量发布
- **监控告警**：设置错误率告警阈值（>1% 触发告警）

#### 7.2.2 代码回滚
- **版本保留**：保留上一个稳定版本 tag，10 分钟内完成回滚
- **数据库兼容**：数据库变更需支持向下兼容，可快速回退
- **回滚流程**：
  1. 发现问题后立即关闭功能开关
  2. 评估问题严重程度
  3. 如需回滚，执行回滚脚本
  4. 验证回滚后功能正常
  5. 问题修复后重新发布

#### 7.2.3 数据回滚
- **备份策略**：每日备份视频数据表
- **回滚脚本**：准备数据回滚 SQL 脚本
- **验证流程**：回滚后验证数据完整性
