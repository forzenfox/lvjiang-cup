# 瑞士轮 UI 优化方案

> 参考英雄联盟官方世界赛瑞士轮 UI 进行的改版方案

## 一、概述

### 1.1 优化背景

当前项目的瑞士轮 UI 采用四列纵向堆叠布局，与英雄联盟官方世界赛的 UI 设计存在较大差异。通过对比分析，官方方案在信息架构、视觉层级、用户交互等方面更具优势。

**本次优化目标：**
1. 重新设计 UI 以匹配官方赛程页面风格
2. 整合项目现有主题配色（深蓝 + 金色）
3. 实现响应式布局自适应 PC/移动端
4. 强化晋级/淘汰状态的视觉表现

### 1.2 官方 UI 核心特征

| 特征 | 说明 |
|------|------|
| BO1/BO3 双标签分区 | 通过标签切换分别展示 BO1 和 BO3 赛制 |
| 横向流程布局(PC) | 从左到右展示晋级流程，单屏展示完整赛程 |
| 轮次标签切换(移动端) | 横向滚动的轮次标签，单轮次展示 |
| 战绩分组清晰 | 0-0、1-0、0-1 等战绩标签明确 |
| 晋级/淘汰可视化强 | 蓝色背景标识晋级，灰色背景标识淘汰 |
| 比赛卡片紧凑 | 水平布局，队标-比分-队标 |
| 连接线设计 | 晋级路径可视化 |

### 1.3 赛制说明

本方案严格遵循瑞士轮赛制规则：

| 轮次 | 战绩组 | 比赛类型 | 场数 | 晋级/淘汰时机 |
|------|--------|----------|------|---------------|
| 第一轮 | 0-0 | BO1 | 8 场 | - |
| 第二轮 | 1-0, 0-1 | BO1 | 8 场 | - |
| 第三轮 | 2-0, 1-1, 0-2 | BO1 | 8 场 | 3-0 晋级，0-3 淘汰 |
| 第四轮 | 2-1, 1-2 | BO3 | 6 场 | 3-1 晋级，1-3 淘汰 |
| 第五轮 | 2-2 | BO3 | 3 场 | 3-2 晋级，2-3 淘汰 |

## 二、设计规范

### 2.1 页面背景色

为保证视觉一致性，全页面统一使用纯深色背景：

| 区域 | 颜色 | Hex 值 | 说明 |
|------|------|--------|------|
| 页面背景 | 深蓝黑 | `#0F172A` | 全页面统一背景色 |
| 卡片背景 | 深蓝黑（透明） | `rgba(15, 23, 42, 0.8)` | 半透明版本用于叠加场景 |
| 边框色 | 深灰 | `#1E293B` | 分割线和边框 |

**设计决策：**
- 取消原有的渐变背景（`bg-gradient-to-b from-primary to-gray-900`）
- 统一为纯深色背景，简洁专业
- 让内容（比赛信息）更加突出
- 提升整体视觉一致性

### 2.2 主题配色方案

基于项目现有配色体系（深蓝 + 金色主题）：

| 用途 | 颜色名称 | Hex 值 | CSS 变量 |
|------|----------|--------|----------|
| 主色-蓝 | Deep Blue | `#1E3A8A` | `primary` |
| 主色-金 | Gold | `#F59E0B` | `secondary` |
| 背景-深 | Dark | `#0F172A` | `background` |
| 背景-灰 | Gray | `#374151` | `muted` |
| 文字-主 | White | `#FFFFFF` | `foreground` |
| 文字-次 | Light Gray | `#94A3B8` | `muted-foreground` |
| 边框 | Border | `#1E293B` | `border` |
| 晋级色 | Qualified Blue | `#1D4ED8` | `qualified` |
| 淘汰色 | Eliminated Gray | `#475569` | `eliminated` |
| 获胜高亮 | Winner Gold | `#FBBF24` | `winner` |

## 九、交互设计

### 9.1 比赛卡片点击

- 点击比赛卡片：弹出比赛详情 Modal
- Modal 内容：完整比分、各小局详情、比赛时间、直播链接

### 9.2 晋级/淘汰队伍点击

- 点击队伍 Logo：弹出队伍详情
- 队伍详情：队名、Logo、选手列表、战队口号

### 9.3 移动端轮次标签切换

- 点击轮次标签：切换到对应轮次的比赛列表
- 支持左右滑动切换轮次
- 当前选中轮次底部有金色(#F59E0B)下划线指示器

### 9.4 横向滚动交互（PC端）

- 宽屏下支持鼠标滚轮横向滚动
- 支持触控板横向滑动
- 可选：添加左右滚动箭头按钮

## 十、技术实现要点

### 10.1 响应式布局实现

```tsx
// SwissStage.tsx
export const SwissStage: React.FC = () => {
  const isMobile = useMediaQuery('(max-width: 1023px)');

  return (
    <div className="w-full">
      {isMobile ? <SwissStageMobile /> : <SwissStagePC />}
    </div>
  );
};
```

### 10.2 组件拆分策略

1. 将 `SwissStage.tsx` 拆分为 PC 和 Mobile 两个版本
2. 每个组件职责单一，便于测试和维护
3. 使用 React.memo 优化渲染性能

### 10.3 配色变量定义

```typescript
// theme/colors.ts
export const swissTheme = {
  qualified: {
    bg: '#1D4ED8',      // 晋级背景
    contentBg: 'rgba(29, 78, 216, 0.2)',  // 晋级内容背景
  },
  eliminated: {
    bg: '#475569',      // 淘汰背景
    contentBg: 'rgba(71, 85, 105, 0.2)',  // 淘汰内容背景
  },
  winner: '#FBBF24',    // 获胜高亮金色
  connector: '#F59E0B', // 连接线金色
  border: '#1E293B',    // 边框色
  textMuted: '#94A3B8', // 次要文字
};
```

### 10.4 响应式策略

1. 使用 Tailwind 的响应式前缀（sm:、md:、lg:、xl:）
2. 断点：1024px 作为 PC/移动端的分界线
3. 移动端优先设计，逐步增强

### 10.5 性能优化

1. 比赛卡片使用虚拟列表（如果比赛数量 > 20）
2. 图片懒加载
3. 动画使用 CSS 替代 JavaScript
