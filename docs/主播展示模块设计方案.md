# 驴酱杯主播展示模块设计方案

## 1. 设计背景

为了提升驴酱杯赛事的影响力和观众参与度，需要在主页面添加一个专门的模块，用于展示参赛的明星主播和邀请的嘉宾主播。该模块将展示主播的海报、个人简介以及直播间跳转入口，方便观众了解主播信息并直接进入直播间观看。

## 2. 设计目标

- **提升用户体验**：为观众提供直观、美观的主播信息展示
- **增强互动性**：提供直播间直接跳转功能，方便观众快速进入直播
- **保持一致性**：与现有页面风格保持统一，确保整体视觉协调
- **响应式设计**：适配不同设备屏幕尺寸

## 3. 数据结构设计

### 3.1 主播数据模型

```typescript
// 主播类型

  id: string;
  nickname: string;       // 主播昵称
  avatarUrl: string;      // 头像URL

  gameId?: string;        // 游戏ID
  level?: PlayerLevel;    // 实力等级
}
```

### 3.2 API 接口设计

需要新增以下API接口：

- `GET /api/streamers` - 获取所有主播列表
- `GET /api/streamers/:id` - 获取单个主播详情

### 5.2 服务层集成

新增 `streamerService.ts` 文件，提供主播数据获取功能：

```typescript
import axios from './axios';
import type { Streamer } from './types';

export const streamerService = {
  async get(): Promise<Streamer[]> {
    const response = await axios.get('/api/streamers');
    return response.data.data;
  },
  
  async getById(id: string): Promise<Streamer> {
    const response = await axios.get(`/api/streamers/${id}`);
    return response.data.data;
  }
};
```

## 6. 样式设计

### 6.1 颜色方案

- 主色调：保持与现有页面一致的紫色系
- 明星主播标签：黄色 (#F59E0B)
- 嘉宾主播标签：紫色 (#8B5CF6)
- 卡片背景：半透明黑色 (bg-white/5)
- 边框：半透明白色 (border-white/10)

### 6.2 响应式设计

- 移动端：单列布局
- 平板：双列布局
- 桌面：三列或四列布局

### 6.3 动画效果

- 卡片悬停：轻微上浮和缩放效果
- 海报悬停：轻微缩放效果
- 标签切换：平滑过渡效果

## 7. 交互设计

### 7.1 标签切换

- 点击标签切换显示不同类型的主播
- 标签状态有明显的视觉区分

### 7.2 卡片交互

- 鼠标悬停时卡片轻微上浮
- 点击卡片打开详情模态框
- 点击"进入直播间"按钮在新窗口打开直播间

### 7.3 模态框交互

- 点击卡片或"详情"按钮打开模态框
- 点击模态框外部或关闭按钮关闭模态框
- 模态框内提供直播间跳转按钮

## 8. 性能优化

- 使用骨架屏减少加载等待感
- 图片懒加载
- 数据缓存
- 分页加载（如果主播数量较多）
