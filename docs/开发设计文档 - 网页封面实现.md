## 1. 概述

### 1.1 产品目标

为驴酱杯赛事网站添加仿 LPL 官方网站的网页封面效果，提升品牌感和视觉冲击力。

**核心体验：**
- 用户访问首页时展示全屏封面动画
- 通过滚动或触摸操作触发退出动画
- 封面退出后展现网站主内容

### 1.2 技术选型

| 技术 | 方案 | 说明 |
|------|------|------|
| **动画库** | Framer Motion v12.34.5 | 项目已安装，支持硬件加速 |
| **样式方案** | Tailwind CSS + 自定义 CSS | 与现有项目一致 |
| **响应式断点** | 1024px | Tailwind `md` 断点 |
| **图片资源** | 临时使用 LPL 官方图片 | 上线前需替换为自有版权图片 |

### 1.3 关键设计决策

#### 决策 1：z-index 层级方案 ✅

**问题：** 导航栏 `z-50`，封面需覆盖导航栏

**方案：** 封面使用 `z-[60]`，直接覆盖导航栏

**理由：**
- 实现简单，无需修改现有组件
- 符合"封面覆盖一切"的产品需求
- 与 LPL 官方实现逻辑一致

**层级结构：**
```
封面组件 (z-[60])
  ├─ 上半部分 (start-t)
  └─ 下半部分 (start-b)

导航栏 (z-50) ← 被封面覆盖
全局 Toast (z-50)
全局 Loading (z-50)
```

#### 决策 2：每次加载显示机制 ✅

**方案：** 不使用 sessionStorage，每次页面加载都显示封面

**理由：**
- 与 LPL 官方网站保持一致
- 简化状态管理逻辑
- 提供更好的视觉体验

#### 决策 3：管理员路径处理 ✅

**方案：** 在 `StartBox` 组件内判断路由，`/admin/*` 路径不渲染封面

**实现：**
```tsx
const isAdminRoute = window.location.pathname.startsWith('/admin');
if (isAdminRoute) return null;
```

 * 触发封面退出动画

const triggerExit = useCallback(() => {
  if (isExiting) return; // 防止重复触发
  
  setIsExiting(true);
  
  // 900ms 后完全隐藏

    setIsVisible(false);

    onExit?.();
  }, 900);
}, [isExiting, onExit]);

    pc: string[];      // PC 端图片

const BackgroundCarousel: React.FC<BackgroundCarouselProps> = ({ isExiting, images }) => {
  const [currentImageIndex, setCurrentImageIndex] = useState(0);
  
  // 3 秒自动切换
  useEffect(() => {
    const interval = setInterval(() => {
      setCurrentImageIndex(prev => (prev + 1) % images.pc.length);
    }, 3000);
    
    return () => clearInterval(interval);
  }, []);
  
  return (
    <motion.div
      className="absolute inset-0 h-full"
      animate={isExiting ? { y: '-100%' } : {}}
      transition={{ duration: 1, ease: 'easeInOut' }}
    >
      {/* 淡入淡出轮播 */}
      {images.pc.map((img, index) => (
        <motion.div
          key={img}
          className="absolute inset-0 bg-cover bg-center"
          style={{ backgroundImage: `url(${img})` }}
          initial={{ opacity: 0 }}
          animate={{ opacity: index === currentImageIndex ? 1 : 0 }}
          transition={{ duration: 1 }}
        />
      ))}
    </motion.div>
  );
};

### 4.2 交互逻辑实现

#### 4.2.1 PC 端 - 鼠标滚轮

```typescript
useEffect(() => {
  if (!isVisible || isExiting) return;

  const handleWheel = (event: WheelEvent) => {
    // 向下滚动触发
    if (event.deltaY > 0) {
      triggerExit();
    }
  };

  window.addEventListener('wheel', handleWheel, { passive: true });
  
  return () => {
    window.removeEventListener('wheel', handleWheel);
  };
}, [isVisible, isExiting, triggerExit]);
```

**关键点：**
- `{ passive: true }`：提升滚动性能
- `deltaY > 0`：仅响应向下滚动
- 防止重复触发：`isExiting` 检查

#### 4.2.2 移动端 - 触摸滑动

```typescript
useEffect(() => {
  if (!isVisible || isExiting) return;

  let startY = 0;

  const handleTouchStart = (e: TouchEvent) => {
    startY = e.touches[0].clientY;
  };

  const handleTouchMove = (e: TouchEvent) => {
    const currentY = e.touches[0].clientY;
    const diff = startY - currentY;

    // 向下滑动超过 50px 触发
    if (diff > 50) {
      triggerExit();
    }
  };

  window.addEventListener('touchstart', handleTouchStart, { passive: true });
  window.addEventListener('touchmove', handleTouchMove, { passive: true });

  return () => {
    window.removeEventListener('touchstart', handleTouchStart);
    window.removeEventListener('touchmove', handleTouchMove);
  };
}, [isVisible, isExiting, triggerExit]);
```

**关键点：**
- 滑动阈值：50px（防止误触）
- 计算逻辑：`startY - currentY > 50`（向下为正）
- 移动端优化：`{ passive: true }`

## 5. z-index 层级管理

### 5.1 当前项目 z-index 分布（重构前）

| 组件 | 文件 | z-index | 说明 |
|------|------|---------|------|
| PC 导航栏 | `Layout.tsx:113` | `z-50` | 顶部固定 |
| 移动端导航 | `Layout.tsx:185` | `z-50` | 底部固定 |
| 全局错误提示 | `Home.tsx:36` | `z-50` | Toast |
| 全局加载 | `Home.tsx:51` | `z-50` | Loading |
| 通用 Modal 遮罩 | `Modal.tsx:39` | `z-[100]` | Portal 渲染 |
| 通用 Modal 内容 | `Modal.tsx:59` | `z-[101]` | Portal 渲染 |
| 确认对话框 | `confirm-dialog.tsx:30` | `z-50` | 与导航栏同层 |
| 选手详情弹窗 | `PlayerDetailModal.tsx:117` | `z-50` | 内联渲染 |
| 队员详情弹窗 | `MemberDetailModal.tsx:41` | `z-50` | 内联渲染 |
| 下拉菜单 | `HeroSelector.tsx:95` | `z-50` | 绝对定位 |
| 队员卡片悬浮 | `MemberCard.tsx:90` | `z-50` | 绝对定位 |
| 拖拽元素 | `Streamers.tsx:91` | `1000` | 临时提升 |

**⚠️ 问题发现：**
- 8 个组件共享 `z-50`，无优先级区分
- 通用 Modal 使用 `z-[100/101]`，会**覆盖封面**
- Portal 渲染打破层级控制（渲染到 `document.body`）

### 5.2 重构方案概览

**详细重构方案请查看：** [`z-index层级重构方案.md`](./z-index层级重构方案.md)

**核心设计：**

```typescript
// src/constants/zIndex.ts
export const ZIndexLayers = {
  HIDDEN: -1,           // 隐藏元素
  BASE: 0,              // 基础层级
  ABSOLUTE: 10,         // 组件内绝对定位元素
  STICKY: 50,           // 导航栏
  DROPDOWN: 60,         // 下拉菜单、悬浮提示
  TOAST: 70,            // Toast 通知
  COVER: 80,            // 网页封面 ✅
  CONFIRM_DIALOG: 90,   // 确认对话框
  MODAL_OVERLAY: 100,   // Modal 遮罩（Portal）
  MODAL: 110,           // Modal 内容（Portal）
  NESTED_MODAL: 120,    // 嵌套 Modal
  DRAGGING: 1000,       // 拖拽元素
} as const;
```

```

├─────────────────────────────────────────┤
│  嵌套 Modal (z-120)                     │

├─────────────────────────────────────────┤
│  确认对话框 (z-90)                      │
├─────────────────────────────────────────┤
│  网页封面 (z-80)                        │ ← 覆盖页面内容，被 Modal 覆盖
├─────────────────────────────────────────┤
│  Toast 通知 (z-70)                      │
├─────────────────────────────────────────┤
│  下拉菜单 (z-60)                        │
├─────────────────────────────────────────┤
│  导航栏 (z-50)                          │ ← 粘性定位
├─────────────────────────────────────────┤
│  组件内绝对定位 (z-10)                  │
│  基础层级 (z-0)                         │
└─────────────────────────────────────────┘
```

### 5.4 封面 z-index 设计决策

**决策：封面使用 `ZIndexLayers.COVER (80)`**

**理由：**
- ✅ 封面需要覆盖导航栏 (50)、Toast (70)
- ✅ 封面**不能**覆盖 Modal (100/110)，否则用户无法操作
- ✅ 80 是合适的值，留出 10 的间隔

**产品逻辑验证：**

| 场景 | 预期行为 | 验证 |
|------|---------|------|
| 封面显示时 | 覆盖页面内容，用户滚动退出 | ✅ z-80 > z-50/70 |
| 封面显示期间触发 Modal | Modal 覆盖封面，用户可操作 | ✅ z-100/110 > z-80 |
| Modal 关闭后 | 封面仍然显示（如果未退出） | ✅ 状态独立 |
| 拖拽元素 + Modal | 拖拽元素在最顶层 | ✅ z-1000 > z-110 |

### 5.5 实现代码

```tsx
// StartBox.tsx
import { ZIndexLayers } from '../constants/zIndex';

<motion.div 
  className="fixed inset-0 bg-black overflow-hidden"
  style={{ zIndex: ZIndexLayers.COVER }}
  initial={{ opacity: 1 }}
  animate={isExiting ? { opacity: 0 } : { opacity: 1 }}
  transition={{ duration: 0.3 }}
>
  {/* 封面内容 */}
</motion.div>
```

**关键点：**
- 使用 `ZIndexLayers.COVER` 常量（值 80）
- `fixed` 定位确保覆盖全屏
- `bg-black` 纯黑背景

### 5.6 边界情况处理

#### 场景 1：封面显示期间触发 Modal

```
时间线：
1. 用户访问首页 → 封面显示 (z-80)
2. 封面显示期间，后台异步加载数据完成
3. 数据加载失败 → 触发错误 Modal (z-100/110)

预期行为：
✅ Modal 覆盖在封面上方
✅ 用户可以关闭 Modal
✅ 封面仍然显示，等待用户滚动退出
```

#### 场景 2：封面退出期间触发 Modal

```
时间线：
1. 用户滚动触发封面退出动画
2. 封面动画进行中（1 秒）
3. 此时触发 Modal

预期行为：
✅ Modal 立即显示
✅ 封面继续退出动画
✅ 两者互不干扰
```

#### 场景 3：Toast 与封面同时显示

```
时间线：
1. 封面显示 (z-80)
2. Toast 通知触发 (z-70)

预期行为：
✅ Toast 被封面遮挡（符合预期）
✅ 封面退出后 Toast 可见（如果未自动消失）
```

### 6.3 实现方案
