# 战队信息扩展功能测试用例

## 1. 文档概述

### 1.1 测试背景
根据《战队信息扩展功能 PRD》文档，需要对战队信息扩展功能进行全面测试，确保功能完整性、正确性和用户体验。

### 1.2 测试目标
- 验证战队图标上传功能的正确性
- 验证参赛宣言功能的完整性
- 验证战队成员信息扩展功能的实现
- 验证管理后台配置功能的可用性
- 确保响应式设计和兼容性

### 1.3 测试范围
- 前端：战队信息展示页面、管理后台配置页面
- 后端：战队信息 API、队员信息 API、文件上传 API
- 数据：战队信息、队员信息、图片上传

### 1.4 测试环境
- 浏览器：Chrome 80+、Firefox 75+、Safari 13+、Edge 80+
- 设备：桌面端、笔记本、平板、手机
- 网络：正常网络环境

## 2. 测试用例

### 2.1 战队图标上传功能

#### 2.1.1 测试用例 ID：TC-TEAM-LOGO-001
**测试项**：支持 JPG/PNG 格式上传
**前置条件**：进入管理后台战队编辑页面
**测试步骤**：
1. 点击战队图标上传区域
2. 选择 JPG 格式图片
3. 确认上传
**预期结果**：
- 图片上传成功
- 显示上传后的图标

#### 2.1.2 测试用例 ID：TC-TEAM-LOGO-002
**测试项**：2MB 大小限制
**前置条件**：进入管理后台战队编辑页面
**测试步骤**：
1. 点击战队图标上传区域
2. 选择超过 2MB 的图片
3. 确认上传
**预期结果**：
- 显示错误提示："图片大小不能超过 2MB"
- 图片未上传

#### 2.1.3 测试用例 ID：TC-TEAM-LOGO-003
**测试项**：分辨率自动压缩处理
**前置条件**：进入管理后台战队编辑页面
**测试步骤**：
1. 点击战队图标上传区域
2. 选择分辨率为 1024x1024 px 的图片
3. 确认上传
**预期结果**：
- 图片上传成功
- 系统自动压缩处理
- 生成缩略图

#### 2.1.4 测试用例 ID：TC-TEAM-LOGO-004
**测试项**：上传失败处理
**前置条件**：进入管理后台战队编辑页面
