# 英雄联盟赛事网站 - 英雄头像展示技术方案文档
## 文档说明
本文档为**英雄联盟赛事前端页面**提供标准化技术方案，实现**根据中文英雄名称 → 自动获取高清英雄头像URL**的完整流程，适配赛事对局信息、选手卡片、英雄池展示等场景，资源稳定、国内访问速度快、无需本地存储图片。

---

## 一、方案核心思路
1. 维护一份**官方标准英雄「英文名-中文名」JSON映射表**（前端可直接引入）
2. 前端通过**中文英雄名**反向查找对应的**标准英文名**
3. 使用**腾讯官方CDN**拼接生成头像URL，直接渲染图片

---

## 三、前端实现步骤（核心）
### 1. 数据预处理（仅执行一次）
将`英文名→中文名`映射转换为`中文名→英文名`，方便查询。

### 2. 工具函数封装
提供**中文英雄名 → 头像URL**的通用方法。

### 3. 页面渲染
直接调用函数获取URL，赋值给`<img src="">`展示。

---

## 四、完整代码实现
### 1. 引入JSON + 封装工具类
```javascript
// 1. 导入英雄映射JSON
import championMap from './lol-champion-map.json';

// 2. 预处理：生成 中文名→英文名 映射（全局仅构建一次）
const championNameToEn = {};
Object.entries(championMap).forEach(([enName, cnName]) => {
  championNameToEn[cnName] = enName;
});

/**
 * 3. 核心工具函数：根据中文英雄名获取头像URL
 * @param {string} cnName - 中文英雄名（例："疾风剑豪·亚索"）
 * @returns {string} 英雄头像URL
 */
export function getChampionIconUrl(cnName) {
  const enName = championNameToEn[cnName];
  if (!enName) {
    // 找不到时返回默认占位图
    return 'https://game.gtimg.cn/images/lol/act/img/champion/Default.png';
  }
  return `https://game.gtimg.cn/images/lol/act/img/champion/${enName}.png`;
}
```

### 2. 页面使用示例（Vue/原生JS通用）
```vue
<!-- Vue组件示例 -->
<template>
  <div class="champion-item">
    <img :src="getChampionIconUrl(heroName)" alt="英雄头像" class="champion-icon" />
    <span>{{ heroName }}</span>
  </div>
</template>

<script setup>
import { getChampionIconUrl } from './lol-champion-utils';
// 赛事数据中的中文英雄名
const heroName = '疾风剑豪·亚索';
</script>

<style>
.champion-icon {
  width: 80px;
  height: 80px;
  border-radius: 50%; /* 圆形头像 */
  object-fit: cover;
}
</style>
```

## 五、适配与优化
### 1. 名称兼容处理
如果你的数据中**只有英雄全名（如“亚索”）**，没有称号，可在映射表中补充别名：
```javascript
championNameToEn['亚索'] = 'Yasuo';
championNameToEn['卡莎'] = 'Kaisa';
```

### 2. 图片样式规范
- 列表展示：`60×60px / 80×80px`
- 选手卡片：`100×100px / 120×120px`
- 样式：`object-fit: cover` + 圆角/圆形，适配赛事UI

### 3. 异常处理
- 中文名称匹配失败 → 返回默认头像
- CDN加载失败 → 配置`onerror`显示占位图

## 六、方案优势
1. **官方资源**：头像标准、无拉伸、无水印
2. **国内极速**：腾讯CDN，访问速度远优于海外源
3. **零维护**：无需下载/上传图片，自动跟随官方更新
4. **极简接入**：前端仅需JSON + 1个工具函数
5. **高适配**：支持原生JS、Vue、React、UniApp等所有前端框架
